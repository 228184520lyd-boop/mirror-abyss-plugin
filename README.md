# Mirror Abyss / 镜渊 1.1.0-alpha.10.5.4

这是基于仓库 `1.1.0-alpha.9.7` 逐阶段修复后的现有功能收尾候选。镜渊仍只围绕五项业务功能运行：

- 执行玩家提示词的正文审核；
- 只保存当前活跃信息的九类表格；
- 阶段性小总结；
- 累计长期大总结；
- 只供玩家查看的关系图谱。

聊天正文始终是原始事实。表格、总结、图谱和世界书都是可以从正文重建的派生数据。任务队列、历史重建、世界书事务和诊断只承担可靠性职责，不参与世界演算。



## alpha.10.5.4 缓存优先重建与任务 UI

- 历史重建不再默认重新读取全部正文；先扫描稳定消息键对应的统一事实包、表格检查点、小总结和大总结。
- 连续有效事实包在本地批量重放，整段缓存只做一次表格/聊天持久化，不产生模型调用。
- 只有缺失、编辑失效、删除关联或无法兼容复用的区段才回读正文并调用 SillyTavern 原生生成接口。
- 完整缓存的手动重建可以做到 0 次模型调用，只重放表格、校验派生数据并重新发布世界书。
- 局部编辑测试中，24 条历史只回读修改的 1 条，另外 23 条复用既有事实；仅在确有长期依赖失效时更新大总结。
- 复用旧事实包时会按当前消息稳定键重映射来源索引，避免删除或插入消息后使用过期下标。
- 有效小总结继续保留；混合阶段总结部分失效时，从已有逐轮摘要恢复阶段记忆，不重新读取未变化正文。
- 任务中心和重建状态条明确显示当前阶段、复用条数、回读正文条数、缓存包数量、模型批次数、进度与错误。
- UI 保持现有工作台布局，只加强任务卡层级、状态图标、进度条、错误区和移动端信息排列。
- 自动测试 100/100；正常游玩审核、表格、小总结、大总结、世界书和图谱语义不变。

## alpha.10.5.3 酒馆原生调用与后台重建

- 所有模型生成统一交给 SillyTavern：当前连接使用 `generateRaw()`，指定配置使用 `ConnectionManagerRequestService.sendRequest()`。
- 镜渊不再实现模型供应商 `fetch`、独立 API 密钥、连接代理、网络重试、限流、超时或熔断。
- 历史重建改成持久后台父任务：点击后立即返回，界面可继续操作；任务中心持续显示批次、范围、进度、耗时、成功与失败。
- 每个历史批次只调用一次酒馆模型，同时输出统一事实、当前表格变化、阶段小总结和沉降凭证。
- 全部批次完成后只调用一次大总结，再统一发布一次世界书。
- 每个成功批次原子更新表格与检查点；失败保留已完成批次，可从失败位置继续。
- 顶部镜渊按钮显示运行中或失败徽标；失败原因不会被 Toast 一闪而过，任务重跑成功后旧红色警报自动清除。
- 修复重建阶段小总结未写入统一事实包，以及最终大总结与重建检查点发生本地提交冲突的问题。
- 120 条历史正文模拟为 10 次批次事实调用＋1 次最终大总结；启动函数约 1.32ms 返回（本地模型模拟，不代表真实 API 推理耗时）。
- 自动测试 97/97；普通 30/100 轮调用结构仍为 69/230，正常游玩语义不变。

## alpha.10.5.2 冻结性能补丁

- 相同 API 地址与密钥即使被多个模型配置引用，也会归入同一脱敏凭据通道，避免表面“双 Profile”实际压满同一上游。
- 同一凭据队列中的前台审核可越过尚未开始的事实、总结与发布请求；已经运行的请求不会被强制中断。
- 统一事实提示词使用裁剪后的活跃表格：保留 ID、标题、内容、状态、关键词、保护标记、生命周期与关系端点，删除更新时间等模型不需要的内部字段。
- 快速短轮仍然合并；只有单次输入超过字符预算时才按体积拆批，历史重建复用同一策略，降低超长请求导致 504 的风险。
- 小总结与大总结在一次派生周期内共享事实包索引，避免长存档反复扫描聊天。
- 批量事实派生只重复持久化区段末条；被后续区段取代的旧派生状态会明确结束，不长期显示排队。
- 普通轮次、阶段总结和长期总结的模型调用次数均未增加。
- 新增 4 项性能回归断言；现有 37 个测试文件全部通过。

## alpha.10.5.1 检测修复

- 修复同一次小总结可删除全部时间地点行的问题；无论沉降计划如何，活跃表格至少保留一个时间/地点锚点。
- 当前事实包只允许 `facts` 驱动九表分发；模型额外输出的冲突 `tableOperations` 不再覆盖代码映射。
- 已存在对象沿用稳定 ID 时，即使模型漏写重复标题，仍可使用旧行标题更新表格。
- 单字人物、物品或地区名称可作为世界书关键词触发，不再产生空关键词条目。
- 多组重复焦点卡分别记录合并来源，避免其他焦点的旧行 ID 串入当前焦点注册表。
- 被更新版本替代的待发布世界书任务会明确结束为“已合并”，不再长期显示排队。
- 已初始化聊天在切换返回后，会检测并恢复因切换取消而未处理的最新正文；从未初始化的旧历史不会自动全量重建。
- 自动测试增至 102/102；普通轮次与阶段节点的模型调用结构未改变。

## alpha.10.5 主要变化

### 可验证的小总结吸收

- 小总结仍只调用原有一次阶段模型，不增加调用。
- 同一次输出增加 `absorbedRowIds` 与 `keepActiveRowIds`：前者证明结果已进入阶段总结，后者明确仍需留在活跃表格。
- 表格行只有在“已结束＋已吸收＋非手工＋非锁定＋未被标记继续活跃”时才能退出。
- alpha.10.4 及更早的小总结仍使用文字吸收兼容回退。

### 对象化大总结

- 大总结不再要求模型自由重写整篇长期文本。
- 模型只输出 `upsert / remove / merge` 操作；代码维护稳定的长期记录并生成最终大总结。
- 未变化的旧记录自动保留；小总结认定的长期候选即使被大总结模型漏写，也不会静默丢失。
- `irreversible` 记录不能被单独 remove 删除，只能更新或合并。
- 旧模型仍返回完整 summary 时走兼容路径，不追加额外修复调用。

### 固定世界书激活映射

- 基础设定固定常驻。
- 焦点、人物、关系、全局、地区、事件、物品、技能和总结固定按关键词触发。
- 开启现有向量开关后，非长期常驻条目同时启用向量；常驻条目不会重复向量化。
- 世界书仍从表格、小总结和大总结确定性重建，不成为独立事实源。

### 调用与体验

- 普通轮次调用结构不变：审核一次、事实提取一次。
- 小总结和大总结只在玩家现有阈值触发，各一次调用。
- 30 轮测试仍为审核 30、事实 30、小总结 6、大总结 3；100 轮仍为 100、100、20、10。
- 快速 10 轮推进仍可合并为一次事实提取，前台不等待总结和世界书。

## alpha.10.4 主要变化

### 两个模型角色

- 审核与必要修正共享前台模型连接。
- 统一事实提取、表格更新、小总结和大总结共享后台记忆模型连接。
- 同一轮必须先确认最终正文，再提取事实；不同连接上的上一轮后台记忆与下一轮前台审核可以并行。

### 正文只理解一次

- 最终正文只进入一次统一事实提取。
- 模型输出本轮结构化事实，代码将事实增量投影到九类活跃表格。
- 小总结只读取统一事实包与当前活跃表格，不重新读取原始正文。
- 大总结只读取上一版大总结、新小总结和当前活跃表格，不重新读取原始正文。

### 恢复真正的小总结和大总结

- 小总结由模型进行阶段压缩，不再把逐轮摘要机械拼接。
- 小总结同时给出安全沉降计划，判断哪些表格行已经不再活跃。
- 大总结由模型维护累计长期记忆，对旧内容执行保留、修改、合并、替代和删除，而不是无限追加文本行。
- 死亡、永久失去、关系断裂、物件销毁等不可逆结果必须先进入小总结，并继续参与长期总结。

### 活跃表格沉降

- `spacetime`、`relationships`、`items`、`skills`、`events`、`regions` 中已结束且被小总结吸收的自动行可以退出活跃表格和世界书。
- 玩家手工行、锁定行、焦点卡、正式人物记录和基础设定不会被沉降器直接删除。
- 小总结没有提及对应对象时，即使模型请求删除，代码也会拒绝并记录为未应用。
- 人物离场通过人物生命周期逐步降级；死亡人物的正式记录不会被自动删除。

### 关系图谱可靠性

- 新关系事实保存明确的 `sourceEntityId` 与 `targetEntityId`。
- 图谱优先按稳定对象 ID 连线，不再依赖关系标题必须同时包含双方姓名。
- 旧存档仍保留名称匹配回退，升级后不会直接断图。
- 图谱仍然只读，不调用模型、不写回表格、不参与正文。

### 保留的稳定性修复

- 快速推进时合并事实提取，所有中间正文仍被处理。
- 世界书只发布追平后的最新状态，并执行回读、缓存、列表和编辑器刷新。
- 历史编辑、删除、Swipe 和分支变化使用批量重建、检查点和失败批次重试。
- 模型漏写本轮未变化的活跃对象时，旧表格记录不会静默消失。
- 焦点别名、改称呼和刷新不会重复创建焦点卡。

## 调用结构

默认阈值取决于玩家设置。以测试中的“小总结每 5 轮、大总结每 2 个小总结”为例，30 轮完整运行是：

```text
审核：30 次
统一事实提取：30 次
小总结：6 次
大总结：3 次
独立修正或复审：0 次
总模型调用：69 次
```

与 alpha.10.3 的本地总结相比，阶段节点增加了总结模型调用，但不会重新发送原始正文。换取的是阶段压缩、长期更新和安全沉降语义恢复。

快速连续推进仍会合并：10 轮突发可合并为 1 次事实提取，再按阈值生成 1 份阶段总结和 1 份长期总结。100 轮本地长流程验证生成 20 份小总结、10 份大总结，并只提交 1 次最终世界书。

## 安装

将部署包内容覆盖到现有 `mirror-abyss-plugin` 扩展目录，然后完全刷新 SillyTavern 页面。不要同时启用两个镜渊版本。

最低 SillyTavern 版本：`1.17.0`。

升级长期存档前，应备份 SillyTavern 数据、聊天文件和相关世界书。本版本仍是 alpha 候选。

## 当前边界

容器环境无法代替真实 SillyTavern 宿主。发布前仍需实机验证：

- 真实模型对事实包、小总结和大总结结构的遵循程度；
- 100 回合以上长存档中的表格沉降和长期记忆质量；
- 移动端后台冻结、页面回收、弱网和真实 504；
- 多聊天、多标签和不同 SillyTavern 小版本的世界书编辑器刷新；
- alpha.9.7/alpha.10.2 长存档升级后的旧关系端点回退效果。

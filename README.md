# Mirror Abyss / 镜渊 1.1.0-rc.4

这是镜渊的产品收尾候选版。核心能力继续使用 `1.1.0-rc.3` 已验证的部署包，新增薄启动层与样式层，不重写事实、总结、世界书和聊天隔离核心。

## 当前产品契约

镜渊把已经沉降的表格、事件总结、长期总结、图谱输入和托管世界书视为当前世界状态。旧正文被编辑、删除或 Swipe 时，不回读整段历史、不重新调用模型，也不暂停后续记忆；后续只处理新消息。

处理顺序固定为：

1. 前台：正文审核与必要修正。
2. 事实关键后台：事实提取和活跃状态更新。
3. 派生后台：事件小结和长期沉降。
4. 发布：世界书事务、回读与冲突保护。

派生总结不得阻塞新的事实提取。概念层只约束结构、状态流和命名，不直接硬编码剧情或扩大提示词。

## rc.4 收尾内容

- 使用 `finalizer.js` 作为薄启动入口，继续加载原 `index.js` 核心。
- 隐藏已停用的历史重建公开恢复入口，公开策略统一为“持久状态权威、仅处理后续消息”。
- 工作区新增运行结构条，明确前台、事实关键后台、派生后台和世界书发布层。
- 优化手机全屏布局、触控尺寸、表单字号、卡片密度、横向标签和无障碍焦点。
- 保留 rc.3 的优先级调度、每 Profile 有界并发、派生任务单槽、流式聚合、首数据计时、聊天与 revision 隔离。
- 修正文档、构建信息和部署清单之间的版本与产品策略不一致。

## 安装

在 SillyTavern 扩展安装界面使用本仓库对应分支地址。更新后完全刷新页面，不要同时启用两个镜渊版本。

最低 SillyTavern 版本：`1.17.0`。

升级长期存档前仍建议备份聊天文件和相关世界书。`rc.4` 是实机收尾候选，不代表已经完成所有手机型号和网络环境测试。

## 验收重点

- 新正文能够先完成前台显示，后台总结不拖住下一轮。
- 新事实提取可以越过仍在排队的长期总结。
- 切换聊天后旧任务不会写入新聊天。
- 手机端工作区可完整滚动，按钮可触控，输入框不会触发异常缩放。
- 世界书发布失败不会破坏本地已提交事实与总结。
- UI 不再出现历史重建启动或重试入口。

结构约束见 `PRODUCT_CONTRACT.md`，构建说明见 `BUILD_INFO.json`。

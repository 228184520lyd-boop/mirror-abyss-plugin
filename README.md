# Mirror Abyss / 镜渊

SillyTavern 的结构化状态表、规则审核、分层总结、关系图谱与聊天世界书发布扩展。

当前版本：`1.1.0-alpha.4`。

## 本版定位

`alpha.4` 完成了对象语义世界书、人物生命周期和总结沉降的第一轮实现。它仍是测试版，建议先在测试聊天运行 10～20 回合，再用于长期存档。

最低 SillyTavern 版本：`1.14.0`。

## 安装

仓库根目录至少直接包含：

```text
manifest.json
index.js
style.css
settings.html
```

在 SillyTavern 中打开“扩展 → 安装扩展”，粘贴 Git 仓库地址。更新后完全关闭并重新打开酒馆页面，只启用一个镜渊版本。

## 首次使用

1. 在“扩展设置”找到“镜渊”，打开控制中心。
2. 在“模型与设置”中，为规则审核、状态表、小总结和大总结选择当前聊天连接或 Connection Profile。
3. 第一次测试先关闭规则审核、向量和世界书同步，只开启自动状态表。
4. 正常生成一条 AI 正文，检查正文下方状态条和状态表。
5. 表格稳定后再开启世界书同步，并在“世界书”页选择“对象语义模式”。
6. 从旧版本升级时，点击“按对象清理并重新发布”。插件只替换自己管理的条目。

镜渊不保存 API Key。密钥由 SillyTavern 的连接与 Connection Profile 管理。

## 运行管线

每条 AI 正文只创建一个唯一任务：

```text
聊天ID + 消息身份 + 正文指纹 + 管线版本
```

处理顺序：

```text
规则审核 → 状态提取 → 小/大总结沉降 → 世界书同步
```

各阶段分别保存。世界书同步失败时，只重试同步，不会重复调用审核和状态模型。

## 九类状态表

- 当前焦点
- 时间与地点
- 人物
- 关系
- 物品
- 技能与能力
- 事件与流程
- 区域状态
- 基础设定

玩家可以新增、编辑和删除表格行。玩家编辑后的行会标记为“玩家锁定”，自动整理和自动沉降不得覆盖或删除；修改后如果已开启世界书同步，会自动重新发布。

普通物品按所有者聚合为“人物名｜物品与资源”；具有独立身份、状态或因果的重要物品单独建行。技能按主体聚合为“人物名｜技能与能力”。

## 人物生命周期

焦点与正式人物拥有独立的生命周期字段：

```text
存在状态：存活 / 死亡已确认 / 存在未知 / 失踪 / 身份存疑 / …
活跃状态：当前在场 / 当前相关 / 离场但仍活跃 / 休眠 / 长期休眠 / 已归档
记忆状态：广泛记得 / 部分人物记得 / 仅记录留存 / 仅痕迹留存 / …
证据等级：已确认 / 可靠记录 / 多方陈述 / 单方陈述 / 推测 / 未知
可能回流条件
阻止回流条件
```

离场不等于死亡；长期未出现不允许推定死亡；被遗忘只改变记忆状态；死亡只终止主动行动，不删除遗留关系、物品、流程和后果。

模型漏写正式人物时，插件会保留旧人物记录。同名人物会复用旧稳定 ID，避免重新建立重复人物。

## 对象语义世界书

默认按现实对象建立世界书条目：

```text
MA｜基础设定
MA｜全局态势
MA｜焦点｜人物名
MA｜人物｜人物名
MA｜关系网络
MA｜区域｜区域名
MA｜事件｜事件名
MA｜流程｜流程名
MA｜物品与资源｜所有者
MA｜物品｜重要物品名
MA｜技能与能力｜主体
MA｜小总结｜当前周期
MA｜大总结｜长期沉降
```

世界书是模型注入发布层；完整表格、历史版本、总结和任务记录仍保存在镜渊本地数据层。

## 小总结与大总结

小总结负责阶段沉降：

- 把已经结束的事实写入小总结；
- 只允许自动移除已完成、关闭、失效、被替代或归档的时空、事件和区域行；
- 不自动删除焦点、人物、关系、物品、技能或基础设定；
- 人物只能按“离场但仍活跃 → 休眠 → 长期休眠 → 已归档”逐级推进；
- 当前在场、当前相关、玩家锁定人物不会被自动归档。

大总结采用累计内推：上一版大总结与新一批小总结合并成新的长期沉降。已被大总结吸收的小总结仍保留在本地历史，但不再占用“当前周期”世界书条目。

## 关系图谱

图谱直接读取当前状态表，不额外调用模型。人物节点会区分：

- 当前人物
- 失踪或身份未知
- 休眠与长期休眠
- 已归档
- 死亡已确认

图谱是可视化辅助层，世界书中的“关系网络”条目是模型读取层。

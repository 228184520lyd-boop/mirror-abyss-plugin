# Mirror Abyss / 镜渊

SillyTavern 的结构化状态表、规则审核、分层总结与聊天世界书发布扩展。

当前版本：`1.1.0-alpha.2`。

这是从 `1.0.0-rc.3.x` 单文件原型重新构建的第一代模块化版本，不是旧代码继续叠补丁。它采用单消息唯一任务、分阶段持久化、Connection Profile、localforage 数据层和固定扩展设置入口。

## 版本定位

`alpha.1` 用于验证新架构和真实酒馆兼容性。已完成自动化测试，但尚未经过你的云酒馆长期实机测试，因此暂不建议直接替换唯一的长期存档环境。建议先在测试聊天运行 10～20 回合。

## 安装

仓库根目录至少需要直接包含：

```text
manifest.json
index.js
style.css
settings.html
```

在 SillyTavern 中打开“扩展 → 安装扩展”，粘贴 Git 仓库地址。安装后完全刷新页面。

最低 SillyTavern 版本：`1.14.0`。

## 首次使用

1. 打开“扩展设置”，找到“镜渊”。
2. 点击“打开镜渊控制中心”。顶部表格按钮只是快捷入口，缺失时不影响使用。
3. 在“模型与设置”中，为规则审核、状态表、小总结和大总结分别选择：
   - 当前聊天连接；或
   - Connection Profile，并填写配置名称。
4. 镜渊不会保存 API Key。密钥应先在 SillyTavern 的连接页面配置，再保存为 Connection Profile。
5. 第一次测试时先关闭规则审核和世界书向量化，只保留自动状态表。
6. 正常生成一条 AI 正文。正文下方会显示审核、表格和同步状态。

## 运行管线

每条 AI 正文只创建一个唯一任务：

```text
聊天ID + 消息身份 + 正文指纹 + 管线版本
```

处理顺序：

```text
规则审核 → 状态提取 → 分层总结 → 世界书同步
```

四个阶段分别保存。世界书同步失败时，只需要重试同步，不会再次调用审核和状态模型。

## 状态表

八个分类：

- 当前焦点
- 时间与地点
- 人物
- 关系
- 物品
- 事件与流程
- 区域状态
- 基础设定

表格采用固定表头与边框，支持人工新增、编辑和删除。人工记录带有“手动”标记，并在下一轮自动整理时强制保留。

## 规则审核

审核提示词由玩家填写。失败处理有三种：

- 保留并标红；
- 隐藏，等待人工处理；
- 安全撤回最新 AI 消息。

安全撤回会再次确认：当前聊天、最新 AI 消息与正文指纹。条件不成立时自动降级为隐藏，不直接修改聊天数组。

## 数据层

- `extensionSettings`：少量全局配置。
- `chatMetadata`：当前聊天世界书名称等轻量索引。
- `localforage`：任务日志、总结和聊天级状态。
- 消息 `extra.mirrorAbyssV11`：与具体正文绑定的审核结果和状态快照。
- 世界书：只作为派生发布层，不是唯一数据库。

同一角色的不同聊天使用带聊天哈希的独立世界书名称，避免多个存档互相覆盖。

## 旧版迁移

- 新版使用独立设置键 `mirrorAbyssV11`，避免与旧版同时加载时直接覆盖。
- 首次启动会迁移旧版常用设置。
- 生成新状态表时，可以读取前文 `extra.mirrorAbyss.tableSnapshot` 作为上一份快照。
- 建议只启用一个镜渊版本。旧版确认不再需要后再关闭。

## 已暂缓功能

世界图谱没有进入 `alpha.1`。当前优先级是启动、消息去重、状态表、审核、总结、存储和世界书同步。图谱会在核心管线完成真实环境验证后重新加入。

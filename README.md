# Mirror Abyss / 镜渊

当前稳定版：`1.2.0`　流水线：`ma-pipeline-60`

镜渊是 SillyTavern 的长线叙事记忆扩展。它不接管角色正文生成，而是在正文生成后完成规则审核、必要修正、事实提取、对象状态维护、分层总结、历史恢复和世界书发布。

## 核心流程

```text
角色正文
→ 规则审核
→ 必要时定向修正
→ 状态与事实提取
→ 对象快照更新
→ 小总结 / 大总结
→ 世界书发布
```

模型负责理解自然语言；插件负责对象身份、稳定 ID、去重、版本链、事务提交、聊天隔离和世界书写入。

## 安装

1. 下载 GitHub Releases 中的 `Mirror-Abyss-1.2.0.zip`。
2. 在 SillyTavern 扩展管理中安装压缩包。
3. 刷新 SillyTavern。
4. 打开“镜渊”工作区，配置审核、状态、小总结和大总结使用的当前连接或 Connection Profile。

最低 SillyTavern 版本：`1.16.0`。

## 稳定版重点

- 固定文本模型协议，不要求模型返回 JSON 或稳定 ID。
- 全局对象短目录 + 少量相关旧快照工作副本。
- 同聊天、同表、同规范名称只有一个对象身份。
- 小总结只维护近期连续性；大总结只固化跨阶段长期事实。
- 小总结和大总结提示词按五个模块公开编辑，无身份型隐藏前缀。
- 表头设置只显示“表头名称｜记录要求”，内部字段键和类型由插件维护。
- 历史编辑、删除和 Swipe 后从最早失效点重建，并支持失败位置续跑。
- 聊天切换永久阻止旧活动任务迟到提交。
- 桌面和 360–430px 手机使用同一套顶部导航工作区。

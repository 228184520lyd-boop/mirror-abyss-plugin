# Mirror Abyss / 镜渊

SillyTavern 的结构化状态表、分层总结、规则校正与聊天世界书插件。

当前版本：`1.0.0-rc.3.1`（候选测试版）

## 安装

在 SillyTavern 中打开“扩展 → 安装扩展”，粘贴你的 Git 仓库地址。仓库根目录必须直接包含：

```text
manifest.json
index.js
style.css
```

## 使用入口

安装并刷新后，酒馆顶部、用户／Persona 等控制按钮附近会出现“表格”图标。点击进入镜渊控制台。

扩展设置页只保留启用开关和“打开镜渊控制台”按钮。右下角浮动“渊”按钮默认关闭，可在“高级”页重新开启。

## rc.3 主要功能

- 顶部统一控制台：总览、独立模型、表格与总结、规则校正、世界书与图谱、高级工具。
- 独立第二模型：用户在插件内填写 API 地址、密钥和模型；支持 OpenAI 兼容接口、Gemini 与 Anthropic。
- 可选回退：也可使用 SillyTavern Connection Profile，或跟随当前聊天连接。
- 规则校正：玩家自行填写审核提示词；新 AI 正文先审核，再进入状态整理。
- 审核结果必须为 `MA_OK` 或 `MA_FAIL`。未通过时可自动撤回最新 AI 消息，或仅标记失败。
- 单事件处理：移除 `GENERATION_ENDED` 的重复触发，只以 `MESSAGE_RECEIVED` 启动新消息管线，并加入消息级 Promise 与正文指纹去重。
- 八类状态表：焦点、时空、人物、关系、物品、事件与流程、区域、基础设定。
- 工作簿式表格：八个分类标签共用一张传统有边框表格，带序号、对象、当前记录、状态与关键词、操作列。
- 普通表单编辑：逐行新增和编辑不再要求玩家直接修改 JSON。
- 明确状态：规则校正、表格整理、世界书同步分别显示等待、处理中、成功或失败。
- 同步失败会在正文面板和控制台中显示具体原因，并提供重新同步按钮。
- 小总结默认每 15 个有效状态回合生成；大总结默认每 6 份小总结生成，两者可自行调整或手动触发。
- 自动创建或绑定聊天世界书，并按状态行、总结和当前连续性核心进行差异同步。
- 独立全屏 3D 世界图谱；3D 依赖加载失败时自动回退为可点击列表。
- 支持旧版快照导入、旧 MA 世界书迁移和当前聊天镜渊数据导出。

## 首次配置

1. 点击酒馆顶部的镜渊按钮。
2. 打开“独立模型”。
3. 选择接口类型，填写 API 地址、密钥和模型。
4. 点击“测试独立模型”。
5. 打开“规则校正”，粘贴需要检查的硬性规则；不需要校正时保持关闭。
6. 打开“表格与总结”，设置小总结回合数和大总结阈值。
7. 正常生成一条 AI 正文。处理顺序为：规则校正 → 状态表 → 自动总结 → 世界书同步。

## 独立 API 注意事项

插件从浏览器直接调用独立 API。部分官方接口会阻止浏览器跨域访问；此时会显示 CORS 错误。处理方式是使用允许跨域的 API 反向代理，或将调用方式切换为“连接配置”。

API 密钥保存在当前 SillyTavern 用户的扩展设置中。共享账号或不可信服务器不应保存私人密钥。

## 规则校正与撤回

- 只自动撤回最新一条 AI 消息。
- 审核接口超时、断网、格式错误时不会自动删除正文，只标记“校正异常”。
- 选择“保留消息并标记失败”时，失败正文不会进入状态整理和世界书。
- 撤回后，失败原因保存在当前聊天的镜渊元数据中，并显示在控制台总览。

## 调用次数

正常开启规则校正时，一轮 AI 正文通常包含：

1. 一次规则校正调用。
2. 一次状态表调用。
3. 达到阈值时，额外一次小总结或大总结调用。

状态表返回无效 JSON 时会进行一次结构修复重试。这是异常恢复，不是事件监听重复调用。

## 数据规则

- 每条 AI 正文的镜渊数据保存在该消息的 `extra.mirrorAbyss` 中。
- 世界书是状态表与总结的派生注入层，不是独立真相源。
- 删除正文后，对应附属状态和世界书派生条目会在下一次同步中清理。
- Swipe 或编辑 AI 正文后，插件使用新正文指纹重新校正和整理；旧异步结果由 revision 与指纹共同拦截。

## 设计与许可证

表格和控制台采用常见的 SillyTavern 扩展交互模式：顶部入口、分栏设置、状态徽标、移动端全屏、传统可编辑表格。Amily2 仅作为工作流与可读性参考；其源码、CSS、图标和界面实现均未复制或改作。

Mirror Abyss 采用 AGPL-3.0。上游设计与实现来源见 `UPSTREAM_SOURCES.md` 与 `THIRD_PARTY_NOTICES.md`。


## 控制台打不开时

rc.3.1 会同时提供顶部按钮、扩展设置按钮和默认开启的右下角“渊”按钮，三者均打开镜渊控制台。若控制台内部渲染失败，会直接显示错误原因。开发者控制台也可执行 `window.MirrorAbyss.openControlCenter()` 作为紧急入口。

# Mirror Abyss / 镜渊

镜渊是 SillyTavern 前端扩展，负责规则审核、九类结构化状态、分层总结和聊天世界书发布。

## 触发边界

- 自动链只接收明确的、非系统、非玩家、且正文非空的 AI/角色消息。
- 玩家消息、系统消息、空消息、来源不明的事件，以及聊天切换后的旧延迟事件不会触发模型调用。
- 编辑、删除或切换旧 swipe 后，派生链暂停，等待用户选择起点并顺序重算。

## 调度与 API

- 当前聊天连接使用 SillyTavern `generateRaw`。
- Connection Profile 使用 `ConnectionManagerRequestService`，不切换全局连接。
- 插件不保存 API 地址或密钥，不直接连接模型供应商。
- 插件任务保持单执行器：不会无界并发，也不会在活动请求中途强行抢占。
- 排队任务按优先级执行：正文审核/修正/状态高于世界书发布，小总结和大总结最低。排队中的大总结不会挡在新正文状态之前。
- 同一连接通道最多一个模型请求；不同原生连接通道只在绕过主流水线的人工连接测试等场景下彼此独立。
- 当前版本保持非流式结构化调用。诊断页记录任务排队耗时、执行耗时、模型通道等待和请求耗时，不记录提示词、正文、响应全文或密钥。

## 数据顺序

1. 审核；任何审核失败在选择“定向修正”时都会先生成最小修正版并复审，复审通过后原位替换当前正文。
2. 一次状态任务产出标准化事实包和九类状态快照。
3. 本地事实与状态先提交。
4. 小总结、大总结和世界书发布作为可恢复的派生阶段分别排队。

总结或世界书失败不会抹掉已经成功提交的事实与状态。历史重算也区分核心重算与派生发布，避免把部分成功误报为数据全部失败。

## 开发验证

```bash
npm install
npm run verify
```

rc.13 在 rc.12 的定向修正恢复基础上，补齐修正进度发布：审核阻断后会先把“修正处理中”保存并推送到消息面板和已打开的工作区，再发起修正模型请求。

rc.14 修复人工整理入口：点击“整理最新正文”或“重试定向修正”时会明确强制定向修正，不再受旧的标红/隐藏设置阻断；人工任务与已在运行的自动任务使用不同去重键，但仍由单执行器顺序执行。

可安装包只需要根目录中的 `manifest.json`、`index.js`、`index.js.map`、`style.css`、`settings.html` 和说明文档。

rc.15 修复“重新生成最新正文 → 点击重新整理正文”的真实操作链：最新一条 AI 正文的 swipe 只废弃该正文旧版本的 artifact、相关总结和旧快照指针，不再创建整段历史失效；旧历史 swipe 仍按原规则要求重算。人工重新整理会清除 rc.14 遗留的最新 swipe 假失效，并接管当前正文尚未开始的自动任务。迟到的最新 swipe 事件不会取消已经针对新正文运行的定向修正。

rc.16 修复“整理中删除最后正文 → 重新生成”的路径。SillyTavern 删除事件只提供删除后的聊天长度，因此镜渊改为检查剩余 artifact：仍一致时只清理已消失的末端整理数据并取消旧请求；出现错位时才进入历史重算。人工整理也能恢复 rc.15 遗留的无起点删除假失效。


rc.17 对 UI 与业务边界做克制收拢，不改变审核、状态、总结和世界书语义：审核/修正层只发布展示状态，不再直接操作消息 DOM；启停由统一运行控制服务负责；手工状态行的增删改由统一提交服务负责。UI 打开、关闭和切换标签仍然只读取状态，不会触发模型。

# Mirror Abyss / 镜渊 1.1.0-rc.4.2

这是 `rc.4.1` 之上的总结安全批次覆盖包，处理两类问题：

1. 保留 `factIndex.byId` 缺失修复，避免小总结出现 `undefined (reading 'get')`。
2. 降低小总结输入规模，减少模型复制旧来源 ID、改写 ID 或生成非本批 ID 的概率。

## 新参数

- 小总结默认触发：`7` 回合。
- 小总结单批总预算：`5,000` 字符。
- 大总结默认触发：累计 `4` 个已提交小总结。

预算包含协议预留，因此真实发送的基础输入通常略低于 5,000 字符。超过预算的已保存失败批次会通过现有检查点恢复逻辑重新拆分，不需要清缓存。

## 来源 ID 边界

小总结提示词现在会在证据正文之前单独列出：

- 本批允许的 `source_fact_ids`
- 本批允许的 `source_row_ids`

模型只能逐字复制这份白名单。既有事件中的旧来源 ID、上下文锚点 ID、对象名称或自行改写的 ID 都不能写入。

严格校验仍然保留。插件不会通过放宽边界，把错误来源写进事件注册表或长期记忆。

## 旧设置迁移

首次加载 `rc.4.2` 时：

- 旧默认 `15` 回合自动改为 `7`。
- 旧默认 `6` 个小总结自动改为 `4`。
- 用户已经改成其他数值时保持原值。

迁移只修改设置，不删除表格、小总结、大总结、事件、世界书或任务记录。

## 安装

这是覆盖包，不是独立完整包。现有插件目录必须保留 `index.js` 与 `style.css`。将本包文件覆盖到现有插件目录后完整刷新 SillyTavern。

详细步骤见 `PATCH_INSTALL.md`。

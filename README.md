# Mirror Abyss / 镜渊

当前稳定版：`1.3.12`　流水线：`ma-pipeline-75`

镜渊是 SillyTavern 的长线叙事记忆扩展。它不接管角色正文生成，而是在正文生成后完成规则审核、必要修正、事实提取、对象状态维护、分层总结、历史恢复和世界书发布。

## 核心流程

```text
角色正文
→ 规则审核
→ 必要时定向修正
→ 状态与事实提取
→ 对象快照更新
→ 小总结 / 大总结
→ 世界书发布
```

模型只负责从正文中忠实提取明确事实；插件负责对象身份、稳定 ID、去重、属性分层、增量结算、硬字数上限、版本链、事务提交、聊天隔离和世界书写入。

## 安装

1. 下载 GitHub Releases 中的 `Mirror-Abyss-1.3.12.zip`。
2. 在 SillyTavern 扩展管理中安装压缩包。
3. 刷新 SillyTavern。
4. 打开“镜渊”工作区，按照总览中的开箱检查完成连接与自动化设置。

最低 SillyTavern 版本：`1.16.0`。


## 1.3.12 历史工作流收拢（第一阶段）

- 新增独立 `workflow` 层，统一解释历史失效、恢复、失败、部分完成和发布阶段。
- 主流水线、世界书发布和诊断不再分别判断同一历史状态。
- 历史检查点、恢复失败、暂停错误和最终解锁统一由工作流接口推进。
- 本阶段不改事实、对象、总结与世界书正文，也不改变现有 ST 存储格式。

## 1.3.11 单一记忆权威内核

- 生产记忆写入统一由 `memory-state-machine.ts` 编排；对象身份、特殊表规则和生命周期各自只负责一种确定性职责。
- `snapshot.ts` 退回快照应用与只读工具，不再导出生命周期写操作；小总结和大总结只提交覆盖结果，再调用统一状态机结算。
- 删除旧 `sedimentation.ts` 与无人使用的 `summary-policy.ts`，避免旧沉降逻辑和新生命周期同时删除或迁移条目。
- 模型响应中的 `entry_action`、`merge_*` 等旧生命周期字段只作为兼容噪声忽略，不能再改变生产状态。
- `absorbed / retired` 旧墓碑只在存档加载迁移中处理；正常游玩主链只接受 `active / settling`。
- 世界书只读取已提交快照；仅执行旁观者与发布相关性的只读过滤，不创建场景、不合并对象、不修复生命周期。
- 新增 rc74 单一权威合约测试，覆盖越权写入、旧协议噪声、总结结算顺序、世界书停用与恢复。

## 1.3.10 记忆网络内容修复

- 记忆网络固定读取当前聊天最近成功快照，不再因当前消息没有快照而显示空白。
- 旧存档没有事件引用字段时，按内部事实已经明确记录的关联对象恢复事件连线。

## 1.3.9 记忆状态机收拢

- 将“事实进入快照后如何唯一化、更新、待结算、总结覆盖、物理删除并同步世界书”收拢为单向状态机，不再由 `snapshot.ts` 分散承担全部职责。
- 新增统一编排入口：`对象身份 → 保护层恢复 → 特殊表规则 → 生命周期迁移 → 旧数据迁移 → 总结结算`。模型仍只提取明确事实，不获得合并、退出或删除权限。
- 条目生产状态统一为 `active / settling`。`absorbed / retired` 只作为旧存档迁移输入；物理删除不是永久状态，删除后不保留插件墓碑。
- 小总结和大总结必须先在同一事务副本写入事实消费／固化标记，再进行覆盖审计；未覆盖、无事实凭证、受保护或宿主未承接的条目保持待结算。
- 修复模型临时对象 ID 与稳定对象 ID 不一致时，恢复或结算指令无法命中的问题；本轮补丁会先绑定到规范稳定 ID。
- 时空单例、对象身份、生命周期和世界书参与规则分别进入独立模块；正常世界书条目保持启用，待结算条目完全停用，物理删除后不再发布。
- 旧存档迁移保持保守：只有存在明确且已被总结覆盖的事实凭证时，才清理旧墓碑；缺少事实引用的旧条目不会被猜测删除。

## 1.3.8 记忆网络融合

- “事件画像”与原“对象图谱”不再互相替换，统一进入“记忆网络”页面。
- 页面提供“融合视图／事件画像／关系图谱”三种显示方式；默认融合视图同时保留事件时间线与对象关系。
- 事件画像继续从事实、对象和总结实时派生；关系图谱继续读取明确关系、关联对象和关联事件，两者不建立第二份记忆。
- 事件画像会为缺少事件表条目的事件生成只读图谱节点，并按 event_id 将事件与参与对象连线。
- 点击事件卡片可在图谱中定位；点击事件节点时，详情区同时显示事实数量、总结层、待结算数量和明确结果。
- 旧设置升级后默认进入“融合视图＋全局网络”；不修改任何表格、事实、总结或世界书正文。

## 1.3.7 总结结算、时空单例与慢模型容错

- 修正总结后的结算顺序：小总结先标记事实已消费，大总结先标记事实已固化，再执行覆盖审计，待结算条目可以真实改变状态、归并或删除。
- 事件明确结束时，插件会沿 event_id 取回已经关联但本轮没有再次出现的条目，只在该事件局部完成结算，不扫描或发送完整仓库。
- 每个聊天的“当前时空”强制为唯一条目 `spacetime_current`；升级旧存档时自动合并重复时空行并改写引用，不要求重新开档。
- 状态模型协议收敛为 `<MA_TURN>`＋`<MA_CHANGE>`：模型只提取明确事实变化，插件在本地生成内部事实与对象补丁；旧 `<MA_FACT>/<MA_ROW>` 继续兼容解析。
- 新增白盒状态请求上下文、输出 token 与分段字符上限。结构性格式错误只进行一次紧凑修复；502/503/504/超时会按本轮正文分段重试，全部分段成功后才提交。
- 未注册字段、禁用表格、对象歧义等语义错误不会被“修复模型”掩盖，仍然作为硬失败保留检查点。

## 1.3.6 无观点事实书记与增量整理

- 状态模型不再提出 `absorb / retire / restore`、删除、合并、未来价值、伏笔或事件结算意见，只把正文中明确发生、结束、替换或被证实的高信息密度事实转换成固定文本。
- 插件只处理本轮触及对象、同事件关联对象和待结算对象；依据明确终态与固定规则机械生成参与状态迁移，不把完整仓库正文发送给模型。
- “关系图谱”工作区改为“事件画像”：从现有事实、对象、总结和待结算状态实时派生，不新增第二份记忆，也不调用模型。
- 角色对象增加最小属性分层：身份与对象定义、长期与现行事实、当前状态、外观与表现。外观、称呼和单次情绪不得反向改写身份锚点。
- 新增白盒硬字数上限：每种对象表、小总结和大总结均可单独设置。世界书发布按完整事实行裁剪低优先层；总结超过硬上限则拒绝提交，不对事实正文做无规则尾部截断。
- 当前递归和召回结构保持不变，不引入原子事实图遍历。

## 1.3.5 世界书结算与真实删除

- `absorb / retire` 不再创建永久隐藏墓碑。原世界书条目先进入短期“待结算”状态，并在 ST 中 `disable=true`，不参与常驻、关键词、向量或递归召回。
- 并入时，持续结果、事实引用、事件引用与必要旧名称先分发到角色、事件、地点、场景或其他宿主。
- 对应事件结束且小总结或大总结完成覆盖后，插件执行覆盖审计；事实已承接、无未决后续的原物品、临时角色、场景和事件容器会从表格及托管世界书中真实删除。
- 待结算期间若对象重新获得独立价值，可显式 `restore`；人工、锁定、基础保护和焦点条目仍禁止自动退出。
- 升级后，1.3.4 留下的旧 `absorbed / retired` 隐藏墓碑会在后续状态维护中检查：已被总结承接且无活跃事实的旧结构直接清除。
- 本版保留插件运行所需的短期结算元数据，但不再把退出对象作为永久隐藏长期记忆保存。

## 1.3.4 条目生命周期与归并（历史行为）

- 自动条目不再创建后永久占位；失去独立召回价值时，可并入角色、事件、地点、场景、组织或其他合适宿主。
- 宿主新增插件维护的“归并记录”，吸收仍有效的结果、事实引用和事件引用；旧条目名称与关键词会成为宿主触发词。
- 原条目转为隐藏墓碑：不显示在普通表格和图谱中，不再发布世界书，但保留稳定身份以阻止下一轮重新建档。
- 完全失效且没有有效宿主的条目可退出独立记忆；重新获得独立持续价值时，可显式恢复。
- 人工条目、锁定条目、基础保护条目和当前焦点对象不会被自动归并或退出。
- 普通出售、赠送、消耗、损毁、丢弃、事件结束等状态不再等于永久删除：关键或可追踪对象继续独立存在，只有后续价值已被宿主承接时才退出。

## 1.3.3 对象覆盖与场景条目

- 新增独立“场景”表：场景记录本次叙事局面，地点记录可复用空间本体，两者不再互相吞并。
- 状态模型的每个条目新增白盒 `kind` 语义类型；插件在保存前校验角色、全局、物品、场景、地点等归属。
- 自动条目被模型写错表时可按语义本质迁移，并保留稳定 ID、旧基础定义和关联引用；人工或锁定条目不会被自动搬动。
- 物品改为逐件覆盖：所有权、位置、数量、完整性、可用性、隐藏状态、用途与后续可取得性任一建立或改变时，分别记录。
- 当前时空存在而模型漏写场景时，插件会生成最小当前场景条目，避免场景视图再次为空。
- 独立重要物品不再因没有关联角色而被世界书发布过滤。

## 1.3.2 条目类型修复

- 默认表中的“区域”恢复为更直观的“地点”，重要或可复用的地点可独立建档。
- 新增白盒“类型与表格分流”，分类按对象本质与事实作用域判断，不按名字、目标、行动或拟人化写法判断。
- 组织、阵营、政权、机构和群体态势即使像角色一样行动，也进入“全局变化”，不再回退到角色表。
- 已存在对象若模型误选表格，插件会优先沿用原表；条目编辑器可直接修改“归属表格”，保留稳定 ID 后移动。

## 1.3.0 开箱标准

- 默认规则可以直接运行，不要求玩家先学习数据库、稳定 ID 或固定文本协议。
- “规则”页统一管理对象准入、背景排除、证据边界、更新冲突、小总结和大总结提示词。
- 对象准入允许明确“不建档 / 不更新”，降低背景板 NPC、模糊身份和推测内容进入长期记忆的概率。
- 表格用途与表头要求继续白盒可编辑；内部字段键、类型、身份匹配和事务边界保持隐藏并由插件维护。
- 记忆网络把事件画像与对象图谱融合：前者展示时间、事实和结算，后者展示对象关系与事件影响；两者共享同一份派生数据。
- 对象表支持搜索、结果计数和固定关键列；事件画像支持搜索、事件状态、事实数量、参与对象、总结版本和待结算对象浏览。
- 工作区和设置区不使用折叠面板；聊天正文下方仅常驻紧凑状态栏，完整处理详情按需展开。
- 总览提供动态配置进度、缺失项提示和下一步入口；任务状态通过可访问的实时状态区反馈。
- 普通界面使用“规则、发布、诊断”等任务语言；固定协议和完整 system prompt 仅在只读诊断页查看。

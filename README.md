# Mirror Abyss / 镜渊

SillyTavern 的结构化状态表、规则审核、定向修正、分层总结、关系图谱与聊天世界书发布扩展。

当前版本：`1.1.0-alpha.6`。最低 SillyTavern 版本：`1.14.0`。

## 本版定位

`alpha.6` 集中修复多 Connection Profile 切换和结构化输出兼容问题，并为关系图谱加入 50%～250% 缩放。审核、修正、状态表和总结可以使用不同连接配置；每次任务都会等待目标配置完成切换，并在结束后恢复任务前的原连接，包括原本未选择任何 Profile 的状态。

结构输出现在先在本地提取代码块或完整 JSON 对象，并容忍解释文字、思考标签、全角标点和尾随逗号；仍无法解析时，只进行一次“格式修复”，不会重新执行审核或状态提取。失败信息会标明实际使用的连接配置并附带返回片段。

本版仍是测试版。建议先逐个点击五种模型连接的“测试”，再在测试聊天运行 10～20 回合。

## 安装

仓库根目录至少直接包含：

```text
manifest.json
index.js
style.css
settings.html
```

在 SillyTavern 中打开“扩展 → 安装扩展”，粘贴 Git 仓库地址。更新后完全关闭并重新打开酒馆页面，只启用一个镜渊版本。

## 首次设置

1. 在“扩展设置”找到“镜渊”，打开控制中心。
2. 在“模型与设置”中，为规则审核、定向修正、状态表、小总结和大总结选择当前聊天连接或 Connection Profile。
3. 在“规则审核”中填写硬规则，选择“定向修正并原位替换”。
4. 最大自动修正次数建议设为 `1`，并保持“相同违规重复时停止”开启。
5. 初次验证可先关闭世界书同步和向量，只测试审核、修正与状态表。

从旧版升级的用户可能仍保留原来的“保留并标红”设置，需要手动改成“定向修正并原位替换”。

镜渊不保存 API Key。密钥由 SillyTavern 的连接与 Connection Profile 管理。


## Connection Profile 与结构输出

“模型与设置”中的 Profile 现在从 SillyTavern 已保存的连接配置下拉选择，避免手输名称错误。一次任务的事务顺序为：

```text
记录原连接
→ 切换并等待目标 Profile 完成加载
→ 执行模型请求
→ 恢复原连接
```

如果模型在 JSON 前后附加说明、Markdown 围栏或思考标签，插件会优先本地提取。开启“专用 JSON 格式修复”后，只有本地解析仍失败才增加一次调用；这次调用只整理原始返回的格式，不重新判断规则、不重新总结，也不产生新的世界事实。

当最终仍失败时，诊断信息会显示任务名称、使用的连接配置和原始返回片段，便于判断是模型不支持结构输出、Profile 选择错误，还是接口返回了异常内容。

## 审核与定向修正

正常管线：

```text
候选正文
→ 隔离并阻止下一轮生成
→ 结构化规则审核
├─ 通过：直接放行
├─ 可修正：发送具体修改点给修正文模型
│           → 再审核
│           → 通过后原位替换
└─ 阻断：执行玩家选择的失败后处理
→ 状态提取
→ 小/大总结
→ 世界书同步
```

审核结果包含：

```text
pass / revise / block
具体违规规则
正文中的违规证据
每一处应如何修改
必须保留的外部事实
给修正文模型的综合修改指令
```

修正文模型收到原文和这些修改点，但它们通过独立的 `generateRaw` 请求传递，不会作为普通聊天消息写入对话。修正成功后，错误正文在原消息位置被最终正文覆盖；保存到消息元数据中的错误正文副本也会清空。

为防止循环：

- 默认最多自动修正一次，最高两次；
- 同一规则指纹再次出现时立即停止；
- `block` 结果不进行局部修正；
- 修正版未通过前，禁止状态提取、回合计数、总结、世界书和向量发布；
- 审核或修正期间通过 Prompt Interceptor 阻止下一轮生成。

技术边界：SillyTavern 的 `MESSAGE_RECEIVED` 事件触发时，候选正文已经进入当前聊天对象。镜渊会立即隐藏并锁住下一轮生成，且不会让自己的状态、总结或世界书模块读取未通过正文；通过后再原位替换。它不向聊天中插入审核说明消息。

## 运行管线与去重

每条 AI 正文只创建一个唯一任务：

```text
聊天ID + 消息身份 + 正文指纹 + 管线版本
```

消息渲染事件只更新界面，不发起模型调用。各阶段分别保存，单个阶段失败时只重试该阶段。世界书同步失败不会重新调用审核、修正或状态模型。

## 九类状态表

- 当前焦点
- 时间与地点
- 人物
- 关系
- 物品
- 技能与能力
- 事件与流程
- 区域状态
- 基础设定

玩家可以新增、编辑和删除表格行。玩家编辑后的行会标记为“玩家锁定”，自动整理和自动沉降不得覆盖或删除；修改后如果已开启世界书同步，会自动重新发布。

## 人物生命周期

焦点与正式人物拥有独立的存在、活跃、记忆和证据状态，并记录可能回流条件与阻止回流条件。离场不等于死亡；长期未出现不允许推定死亡；被遗忘只改变记忆状态；死亡不删除遗留关系、物品、流程和后果。

模型漏写正式人物时，领域层会保留旧人物记录。同名人物优先复用稳定 ID，避免重复建档。

## 对象语义世界书

默认按现实对象建立世界书条目：

```text
MA｜基础设定
MA｜全局态势
MA｜焦点｜人物名
MA｜人物｜人物名
MA｜关系网络
MA｜区域｜区域名
MA｜事件｜事件名
MA｜流程｜流程名
MA｜物品与资源｜所有者
MA｜物品｜重要物品名
MA｜技能与能力｜主体
MA｜小总结｜当前周期
MA｜大总结｜长期沉降
```

世界书是模型注入发布层；完整表格、历史版本、总结和任务记录保存在镜渊本地数据层。

## 小总结与大总结

小总结负责安全沉降已结束内容。它只允许自动移除符合结束条件的时空、事件和区域行，不自动删除焦点、人物、关系、物品、技能或基础设定。人物只能逐级从离场状态进入休眠、长期休眠和归档。

大总结使用累计内推：上一版大总结与新一批小总结生成新的长期沉降。已经被吸收的小总结保留在本地历史中，但退出当前周期世界书条目。

## 关系图谱

图谱直接读取当前状态表，不额外调用模型。人物节点会区分当前、失踪或未知、休眠、长期休眠、归档和死亡状态。图谱支持按钮、滑杆、恢复 100% 和自动适应窗口，缩放范围为 50%～250%；手机端可在画布中双向滚动。图谱是可视化辅助层，世界书中的“关系网络”条目是模型读取层。

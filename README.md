# Mirror Abyss / 镜渊

SillyTavern 的结构化状态、快照总结与聊天世界书插件。

当前版本：`1.0.0-rc.1`（功能完整候选版）

## 安装

在 SillyTavern 中打开“扩展 → 安装扩展”，粘贴：

```text
https://github.com/228184520lyd-boop/MA
```

仓库根目录必须直接包含 `manifest.json`、`index.js` 与 `style.css`。

## 主要功能

- 每次有效 AI 正文后自动生成八类完整状态快照：焦点、时空、人物、关系、物品、事件与流程、区域、基础设定。
- 玩家输入、AI 正文和当前表格作为混合变量运算；玩家手工修改不单独标注来源，也不默认锁定。
- 状态表显示在对应正文下方，可折叠，可逐行新增、修改、删除。
- 正文与附属表格、总结共用生命周期；删除正文后附属数据消失，不恢复、不重演涌现世界线。
- 小总结默认每 15 个有效状态回合生成；大总结默认每 6 份小总结生成。两者也可手动触发。
- 小总结和大总结采用“快照压缩”，保留当前结果、持续影响与未决事项，不写流水账。
- 自动创建或绑定聊天世界书，并按状态行进行差异同步与清理。
- 焦点、时空、基础设定和当前连续性核心使用常驻条目；其他状态默认关键词触发，可选向量；总结使用向量条目。
- 删除正文或总结后，对应镜渊世界书条目在下一次同步中清除。
- 独立移动端全屏 3D 世界图谱：单指旋转、双指缩放、点击节点编辑；3D 依赖加载失败时自动回退为列表。
- 图谱可显示并编辑当前聊天世界书中非镜渊管理的普通条目。
- 支持表格、小总结、大总结分别指定 SillyTavern Connection Profile 名称；留空时跟随当前连接。
- 支持旧版卡片快照导入，以及 `MA_世界全局 / MA_焦点_* / MA_区域_* / MA_角色_*` 世界书迁移。
- 支持导出当前聊天的镜渊附属数据。

## 首次使用

1. 确认 SillyTavern 已有可正常生成文本的 API 连接。
2. 打开“扩展设置 → 镜渊”。Connection Profile 名称可留空。
3. 保持“每轮自动更新表格”和“自动同步聊天世界书”开启。
4. 正常生成一条 AI 正文。正文下方会出现“镜渊状态”。
5. 点击右下角“渊”按钮打开世界图谱。

## 数据规则

- 每条 AI 正文的镜渊数据保存在该消息的 `extra.mirrorAbyss` 中。
- 世界书是表格与总结的派生注入层，不是独立真相源。
- 删除中间正文不会重新演算后续涌现剧情；当前最新表格继续作为现实状态。
- 若被删除正文属于某份总结的来源，该总结及依赖它的大总结会失效并被清除。
- 重新生成、Swipe 或编辑 AI 正文时，插件以当前有效回复重新整理该条状态；过期异步结果由 revision 拦截。
- 正文模型空跑时不产生状态回合；记忆调用失败时保留正文并显示“未同步”，可点击重试。

## 3D 图谱

默认从 CDN 加载 `3d-force-graph`。图谱完成初始布局后停止持续演算，节点不要求自动旋转；玩家移动的是相机视角。无法访问 CDN 时，图谱自动切换到可点击列表，状态与世界书编辑仍可使用。

## 兼容性与验证状态

本包已完成：JavaScript 语法检查、manifest JSON 检查、文件结构检查和纯函数静态审查。

它尚未在你的云酒馆实例、具体主题、具体 Connection Profile 与移动设备上完成实机回归，因此版本标记为 Release Candidate，而不是宣称零缺陷正式版。首次安装后建议先在测试聊天运行 3～5 回合，再用于长存档。

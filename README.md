# Mirror Abyss / 镜渊

版本：`1.2.0-rc.18`　流水线：`ma-pipeline-21`

镜渊是 SillyTavern 前端扩展。它在角色正文生成完成后提供规则审核、必要的定向修正、事实与九类状态提取、分层总结，以及当前聊天世界书发布；它不生成主叙事正文，不替代 SillyTavern 的连接系统。

## 运行边界

- 自动链只接受明确的非系统角色正文：`is_user === false`、非系统、正文非空。
- 玩家消息、系统消息、来源不明事件、打开 UI、切换标签和聊天切换本身不会发起模型调用。
- 当前聊天连接使用 SillyTavern `generateRaw`；Connection Profile 使用 `ConnectionManagerRequestService`。
- 审核、状态提取和连接测试优先使用 JSON Schema；接口报错或返回 `{}` 时只回退一次无 Schema 请求，不形成调用循环。
- 插件不保存 API 地址或密钥，不切换全局 Profile，不直接请求供应商端点。

## 调度方式

- 全插件仍只有一个业务任务实际执行，不增加无界并发。
- 已经运行的任务不被抢占；待执行任务按价值排序：正文状态高于世界书、小总结和大总结。
- 事实与状态先提交；小总结、大总结和世界书作为后台派生阶段分别执行。
- 同一模型连接通道同时只有一个请求，不同通道可彼此独立。

## 数据安全

- 人工行和锁定行不会被自动状态结果覆盖。
- 编辑、删除或切换旧 swipe 后，旧请求和旧排队任务立即失效。
- 聊天切换、禁用、重启和重置后，旧任务无法继续提交。
- 保存调用绑定发起时的聊天上下文，并在保存返回后再次校验。
- 世界书只操作同时满足镜渊托管标记和完全相同 `chatKey` 的条目。

## 分阶段按钮与世界书刷新

- 最新角色消息下方直接显示“审核正文、重新修正、生成表格、小总结、大总结、同步世界书”六个阶段按钮；工作区总览、审核页、表格页和总结页保留对应入口。
- 审核按钮只运行审核；修正按钮只运行修正与必要复审；表格按钮只在审核通过后提取状态。
- 修正成功后旧表格、旧派生总结会失效，世界书条目先暂停，等待重新生成表格。
- 最新一条 AI 正文被编辑或 swipe 后，用户显式重新整理成功可恢复同步；更早历史仍必须完整重算。
- 状态表成功后，UI 会立即显示总结/同步为“排队”或“跳过”，不再停留在“等待”。
- 世界书保存后先刷新世界书列表，再刷新目标编辑器；目标书正在显示或手动强制同步时，会等待条目 DOM 重绘完成。
- 新建并关联聊天世界书后，会立即更新酒馆的聊天世界书按钮状态，不再依赖退出或切换聊天。

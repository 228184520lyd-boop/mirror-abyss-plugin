# Mirror Abyss / 镜渊 1.1.0-alpha.10.7.4.1

本版本继续以 `1.1.0-alpha.10.6.0` 为稳定基线，完成第二轮记忆层重构。插件的数据链现在固定为：

```text
AI 最终正文
  ↓
正文准入门（审核可关闭）
  ↓
事实提取并更新活跃表格
  ↓
表格变化生成独立事件条目
  ↓
事件条目形成阶段概览并执行安全沉降
  ↓
事件总结与上一版长期记录生成大总结
  ↓
按条目独立采用常驻 / 关键词 / 向量方式发布世界书
```

关系图谱仍位于观察层，只读取当前表格供玩家查看世界，不调用模型、不写回业务数据、不参与正文、事件总结或长期总结。

## 10.7.1 连接错误页诊断修复

当当前聊天连接或指定 Connection Profile 返回 HTML 页面时，插件现在会在进入事实协议解析前中止，并明确显示：

- 出错的是当前聊天连接还是具体 Connection Profile；
- 可安全显示的 API 类型和模型名；
- 能识别时显示 HTTP 状态与 HTML 页面标题；
- 可能属于 Base URL/接口路径、反向代理、鉴权、限流或上游服务故障；
- 该错误不是 `factExtraction` 的纯文本协议、事件条目或 JSON 格式错误。

底层 `Unexpected token '<'` 异常不再与外层诊断重复拼接。非 502/503/504 的 HTML 配置错误也不会在历史重建中无意义自动重试。正文已经通过准入时仍保持正文不受影响，失败的后台任务可在修正连接后重新执行。

## 本版本完成的更新

### 1. 小总结不再是实际记忆发布单元

小总结仍保留标题、阶段概览和沉降说明，供玩家查看处理结果；但真正进入世界书并参与后续更新的是独立事件条目。

每个事件条目可单独维护：

- 稳定事件 ID；
- 已确认事实；
- 涉及人物、组织与地点；
- 发生时间与最后确认时间；
- 活跃、未决、结束、休眠、历史或痕迹状态；
- 传播范围、已知者、传播渠道与明确出现的误传；
- 留下的记录、物证、传言、流程痕迹和持续影响；
- `exact / compressed / fuzzy / trace` 信息精度；
- `critical / high / normal / low` 重要度；
- `constant / keyword / vector` 独立发布方式；
- 来源表格行、来源事实和来源总结。

### 2. 传播与痕迹只从正文事实中提取

插件不会在后台自行模拟传播。已知者、渠道、误传、痕迹和影响必须来自审核后的最终正文及其表格事实。

事件条目不保存“下一次复核时间”。它只保存发生时间和最后确认时间；条目再次被触发时，由正文 AI 对比当前时间判断信息是否过时、模糊或仍然有效。

### 3. 精度只能由新事实重新确认

没有新的来源事实时，旧事件不能从 `fuzzy` 或 `trace` 自动恢复为 `exact`。只有后续正文出现新的明确确认，事件才允许提高精度。

大总结只能降低或保持事件精度，不能把已经沉降掉的姓名、数字、具体时间和地点重新补回。

### 4. 大总结同时维护长期记录与事件沉降

大总结仍只读取：

- 上一版长期记录；
- 本批事件总结；
- 事件总结产生的长期候选。

它不读取原始正文，也不读取当前活跃表格。除长期记录的增删改合并外，大总结可以对本批已结束事件执行：

- 保持；
- 降低精度并改写为更模糊的事实；
- 从关键词触发转为向量触发；
- 在极严格条件下删除低重要度、无持续影响的痕迹条目。

### 5. 发布策略成为条目属性

事件条目不再共用一个全局发布开关：

- `constant`：极少量仍活跃、重要且必须持续注入的事件；
- `keyword`：人物、地点、物品、流程或事件词命中后触发；
- `vector`：模糊历史、风声、痕迹及语义相关记忆。

代码会自动限制常驻滥用，并强制 `fuzzy` 和 `trace` 条目使用向量方式。基础设定仍按原逻辑常驻，活跃表格仍按原布局发布。

### 6. 编辑、删除和历史重建保持可追溯

事件注册表是由小总结中的事件操作和大总结中的事件沉降操作重新推导出的缓存。正文编辑、删除或 Swipe 导致总结失效时，插件会先删除受影响总结，再重建事件注册表，避免残留已经失效的事件条目。

## 保留的稳定能力

- 审核可关闭；关闭时不产生审核 API 请求。
- 审核通过或修正版写回后，最终正文才进入表格。
- 异步任务队列、取消、失败记录、跨标签协调和聊天作用域隔离。
- SillyTavern 当前连接与 Connection Profile 原生调用。
- 九类活跃表格、手工锁定行、生命周期和事实来源。
- 世界书 Outbox、回读验证、冲突保护和恢复日志。
- 历史编辑、删除、Swipe 后的缓存优先重建。
- 前端任务反馈与只读关系图谱。

## 安装

将压缩包中的 `mirror-abyss-plugin` 文件夹覆盖到 SillyTavern 扩展目录，然后完全刷新页面。不要同时启用两个镜渊版本。

最低 SillyTavern 版本：`1.17.0`。

长期存档升级前应备份聊天、相关世界书和 SillyTavern 数据目录。本版本仍为 alpha 开发版本。

### alpha.10.7.2 修复

本版修复 Profile 首次调用串线、世界书状态显示滞后，以及焦点卡被人物表吸收。焦点卡仍是独立高频表格；关系图谱保持只读。


## alpha.10.7.4 运行一致性

世界书状态以服务器回读指纹为准；正文状态条叠加实时任务队列；同一正文自动处理具备幂等保护；事件 ID 在唯一匹配时延续；向量明确不可用时自动降级为关键词触发。


## 10.7.4.1 启动热修复

世界书服务器回读不再位于插件 UI 挂载前。镜渊会先显示入口图标并进入可操作状态，再在后台校验实际世界书。回读慢或失败不会阻止插件启动。

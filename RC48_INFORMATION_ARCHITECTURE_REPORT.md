# rc.48 手机信息架构重构报告

版本：`1.2.0-rc.48`  
流水线：`ma-pipeline-50`

## 实机问题

rc.47 已解决视口高度、安全区和对象表横向溢出，但用户实机截图仍显示两类结构性问题：

1. 每条正文下方默认展开完整五阶段、说明、按钮和排错工具，插件信息长度接近正文；
2. 手机工作区把九个功能入口平铺为三行网格，首屏缺少主次，总览同时展示状态卡、五张阶段卡、排错工具、任务与架构说明。

这不是继续增加媒体查询能够解决的问题，因此 rc.48 调整信息架构，但不修改 pipeline、存储、总结或世界书逻辑。

## 正文状态条

- 默认只显示一行：状态点、`镜渊`、当前流程状态和对象数；
- 成功、等待和运行状态默认折叠完整详情；
- 失败状态首次默认展开，但玩家可手动收起；
- 五阶段进度、继续流程、重试和高级排错只在详情区显示；
- “打开工作区”保留在详情区，旧消息同样可进入对应工作区；
- 面板重新渲染时保留玩家主动展开或收起的选择。

## 手机工作区

- 删除手机顶部九宫格导航；
- 页头只保留菜单、当前页面标题和关闭按钮；
- 九个完整页面收进左侧滑出抽屉，并继续按“运行 / 管理 / 系统”分组；
- 选择页面后自动关闭抽屉；
- 抽屉带遮罩、独立滚动、安全区和关闭按钮；
- 桌面端继续使用常驻左侧栏，不改变桌面功能入口。

## 总览首屏

总览只保留：

1. 当前流程状态；
2. 单一卡片中的五阶段紧凑进度；
3. 对象、总结、世界书三个常用入口；
4. 当前聊天任务列表。

以下内容退出正常总览：

- “本版架构原则”说明长文；
- 单步排错工具；
- 五张独立大阶段卡。

单步阶段工具移动到“诊断 → 高级排错工具”。

## 参考边界

参考 SillyTavern 官方总结插件的成熟交互原则：主要状态和常用动作直接可见，提示词及其他高级控制默认折叠，需要时再展开。镜渊只采用这种信息分层思路，没有复制其业务代码。

## 验证

新增 `tests/rc48-information-architecture.ts`，检查：

- 正文默认使用单行状态条；
- 详情区可折叠且不占空间；
- 手机导航使用抽屉而非多列网格；
- 页面切换后抽屉自动关闭；
- 总览不再显示架构说明和单步排错；
- 高级排错仍可从诊断页进入；
- 桌面侧栏、动态视口、手机对象卡片和全部原业务入口继续存在。

手机尺寸静态预览已经检查关闭菜单与打开菜单两种状态。真实 SillyTavern 中的字体、主题、地址栏和触控仍需实机最终确认。

# 镜渊长期记忆标准 1.0

适用版本：Mirror Abyss `1.3.0` / `ma-pipeline-62`

本标准用于确定镜渊默认提示词、表格、图谱和 UI 的开箱行为。它不是要求玩家照抄论文架构，而是把公开研究中可验证的记忆分层思想，转成适合 SillyTavern 长线叙事插件的确定性边界。

## 1. 设计依据

MemGPT 使用分层记忆与虚拟上下文管理来跨越有限上下文窗口；镜渊据此坚持“不同层承担不同职责”，而不是把完整历史反复塞回主模型。

Generative Agents 将经验记录、较高层反思和动态检索分开；镜渊据此把当前快照、内部事实、小总结、大总结和召回出口分离。

GraphRAG 通过实体关系、层级和摘要支持跨文档的结构化理解；镜渊只借用“关系结构适合整体浏览和关联检索”的部分，不把所有叙事文本强制图谱化，也不让图谱成为新的事实来源。

W3C WCAG 2.2 的相关说明强调输入需要可理解的标签与指引、错误需要文字说明、动态结果需要可感知的状态消息；镜渊据此把规则解释放在输入附近，并使用始终可见的分区与实时状态反馈。

## 2. 五层记忆职责

### 2.1 当前快照

回答：对象现在是什么状态？

只保留当前有效版本。新状态明确替换旧状态时覆盖；未返回字段保持；普通复述不产生修订。

### 2.2 内部事实

回答：什么已发生事实导致当前状态？

每条事实应有主体、变化、结果、来源和证据级别。事实是总结与历史恢复的依据，不直接堆成大量发布条目。

### 2.3 小总结

回答：继续当前事件线的下一步，必须保留什么？

只保留近期连续性，允许较多仍在进行的限制、风险和未决事项，不保留对白与动作流水账。

### 2.4 大总结

回答：近期过程全部遗忘后，什么仍会跨阶段改变未来？

只保留长期且已确认的信息。它必须重新审查资格，不能只是小总结缩写。

### 2.5 关系图谱与世界书

图谱负责人工浏览明确关系；世界书负责向主模型发布和召回。两者都读取内部有效记忆，不反向修改事实资格。

## 3. 写入门槛

对象只有同时满足以下条件才新建：

1. 有稳定可区分的身份；
2. 已形成后续生成必须知道的独立事实；
3. 该事实具有持续关系、因果作用、目标、责任、状态、规则或未决价值之一；
4. 证据来自当前正文或已提供的旧记录，而非常识补全。

姓名、服装、外貌、位置、情绪、一次动作、单句台词、职业称谓和当前场景出现都不是独立准入证据。

## 4. 默认排除

默认不建档：路人、宾客、观众、侍卫、仆从、店员、士兵、村民、围观者、临时通报者、引路者、递交物品者、匿名群体、普通陈设和只服务单次动作的对象。

排除不是永久黑名单。后续正文明确赋予持续身份或因果作用时，可以升级建档。

## 5. 证据与不确定性

- 明确发生：`confirmed`。
- 他人陈述、传闻、主观判断：保留 `reported` 或 `uncertain`。
- 新旧冲突：只有明确推翻、结束或替换时才更新。
- 同名与近名：没有别名、关系或连续上下文证据时不合并。
- 无法确认：不建档或不更新，优先于猜测。

## 6. 更新标准

只记录新增、结束、替换或实质变化。再次出现、普通强调、措辞润色和数组顺序变化不算更新。

基础定义仅在明确重定义、毁灭、重建、转让或人工编辑时改变。当前状态写唯一有效版本；未决事项解决后关闭；同一信息只保留一次。

关系只有在形成承诺、归属、冲突、合作、支配、依赖或其他会影响后续行为的变化时更新。普通同场、普通交谈和单次帮助不建立长期关系。

## 7. 表格标准

每张表同时定义两件事：

- **对象资格**：什么对象属于这张表；
- **字段资格**：每个字段只记录什么。

角色表不接收背景板；物品表只记录有归属、使用、损坏、转移、稀缺或因果作用的物品；事件表必须有可识别的起因、行为、结果、持续影响或未决事项；自定义表必须写清业务边界，不能作为无法分类信息的杂项桶。

普通编辑只暴露“表头名称｜记录要求”。内部字段键、字段类型、稳定 ID 和保护权限不可由提示词接管。

## 8. 图谱标准

图谱连线优先级：

1. 明确关系状态；
2. 明确关联对象；
3. 明确关联事件；
4. 旧记录文本提及兼容回退。

有显式关联时，不再扫描整段文本猜测额外关系。兼容回退边必须有可见标记。图谱不创建、合并或删除对象。

## 9. UI 标准

- 关键规则与操作不放在折叠区。
- 页面按用户任务分配，不按源码模块分配。
- 每个可编辑规则附近给出用途、边界和影响说明。
- 总览显示完成度、缺失项和下一步，不要求用户自行推断配置状态。
- 保存、运行、失败、搜索结果和进度使用实时状态消息。
- 错误同时说明“哪里失败”和“下一步怎么处理”。
- 底层协议、字段键和 ID 只在诊断页只读展示。
- 动效只用于状态、搜索高亮和空间关系提示，并尊重 `prefers-reduced-motion`。

## 10. 默认与高级的边界

开箱默认值必须能在不修改提示词的情况下运行。白盒编辑允许改变信息资格和表达，但不得改变固定文本协议、稳定身份、事务提交、聊天隔离、历史恢复或世界书安全边界。

“恢复标准规则”应恢复经过版本控制的默认值，而不是清空配置。

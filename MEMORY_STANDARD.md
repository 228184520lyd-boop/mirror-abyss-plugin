# 镜渊长期记忆标准 1.1

适用版本：Mirror Abyss `1.3.9` / `ma-pipeline-72`

本标准规定默认提示词、对象表、总结、记忆网络和世界书发布的开箱边界。

## 1. 总原则

镜渊采用持续漏斗，而不是末期全仓清理：

```text
每轮明确事实
→ 当前对象状态
→ 事件近期总结
→ 阶段长期总结
→ 世界书召回包
```

模型是无观点书记，只做事实转换；插件负责身份匹配、去重、结算、压缩、参与状态和删除。

## 2. 信息层职责

### 2.1 当前快照

回答对象此刻的有效状态。只应用正文明确建立的变化；未返回字段保持，复述不产生修订。

### 2.2 内部事实

回答正文明确发生了什么变化，以及变化来自哪条消息和哪条事件线。内部事实是总结与历史恢复原料，不直接大量发布到世界书。

### 2.3 小总结

承接同一事件线近期仍需连续的明确事实。它不记录动作流水账，不补全未知，也不枚举“还有什么没解释”。

### 2.4 大总结

固化已经确认、跨阶段仍成立的结果和变化。它处理已整理材料，不承担全仓扫描、补漏和自由价值判断。

### 2.5 记忆网络

从已有事实、对象和总结派生事件索引。画像不保存第二份正文，不调用模型，也不反向修改记忆。

### 2.6 世界书

世界书是面向主模型的发布与召回出口，不是内部事实账本。退出条目可以先停用，事实被承接后真实删除。

## 3. 书记模型事实边界

允许提取：

- 明确身份或设定的新增、纠正和转化；
- 当前状态、位置、所有权和数量变化；
- 明确关系、能力、阵营和长期属性变化；
- 已经发生的事件结果；
- 正文明确说出的决定、承诺、任务和解除；
- 明确因果和终止事实。

禁止提取或输出：

- 重要性、未来价值和保留建议；
- 动机分析、伏笔识别和背景补全；
- 未被正文建立的未知问题；
- 合并、删除、退休、恢复或结算意见；
- 稳定 ID、事实 ID 和事件 ID。

“高信息密度”由白盒表头和规则定义，不由模型自由评价。

状态主协议只要求模型返回对象字段的明确变化；内部事实、对象补丁、事件 ID 和稳定 ID 均由插件生成。结构性格式问题可以规范化，语义越权不得自动修复。慢网关回退只能拆分本轮正文，不能扩大为仓库扫描。

## 4. 对象准入

只要对象可稳定区分，且它的状态、位置、所有权、数量、限制或事件作用是后续连续性需要的明确事实，就可以建立临时工作条目。建档不等于永久保存。

姓名、服装、外貌、站位、一次情绪、短动作、单句台词和普通职业称谓不足以把背景对象建成角色。场景和可区分物品只要具有当前连续性所需状态，可以暂时建档。

默认排除：无持续作用的路人、宾客、观众、侍卫、仆从、店员、士兵、村民、通报者、引路者、匿名群体、普通陈设和单次动作工具。

## 5. 证据和更新

- 明确发生的内容写入；传闻和主观判断按现有证据等级保留。
- 身份不清、同名无法确认或新旧冲突没有明确解释时，省略优于猜测。
- 只有新增、结束、替换或实质变化才更新。
- 再次出现、普通强调、措辞润色和数组换序不算变化。
- 新版本不要求模型生成未解决事项；旧数据仅兼容读取。

## 6. 角色属性层

### 身份与对象定义

高保护身份锚点。普通外观、称呼、误认、情绪和临时状态不得改写。只有正文明确纠正、重定义、转化、毁灭、重建或人工编辑时变化。

### 长期与现行事实

正文明确成立且持续时间较长的属性或事实。单次行为不能推导长期人格；只有正文明确建立长期改变时才更新。

### 当前状态

正在发生、短期或阶段性的状态。新事实可以替换旧当前值，旧状态的历史意义由事件和总结承接。

### 外观与表现

服装、伪装、称呼、当前外观和他人明确认知。表象不得反向修改身份锚点。

## 7. 每轮增量整理

日常状态提取只向模型提供相关候选短目录和少量高相关工作副本。插件在本地只处理本轮触及对象、同事件关联对象和对应待结算对象。

不得为了大总结把完整仓库正文发送给模型。专用迁移和用户主动一致性检查不属于正常游玩主链。

## 8. 条目结算与退出

模型不决定条目生命周期。插件依据明确事实和固定规则处理：

1. 对象保持独立；
2. 原条目停用并把持续结果分发给宿主；
3. 明确终态、无宿主后果的条目停用；
4. 总结覆盖和保护检查通过后真实删除；
5. 删除前再次被明确使用时恢复。

普通可替代物品在出售、交付、消耗、损毁、丢弃等明确终态后可以结算。唯一、神器、证物、任务、关键和不可替代物品不会仅因所有权转移而退出。角色自动退出保持保守。人工、锁定、基础保护和焦点对象禁止自动退出。

退出是容器消失，不是事实消失。持续结果必须先进入仍活跃对象、事件总结或长期总结。

## 9. 硬字数上限

每个对象表、小总结和大总结的上限均为白盒设置。

- 上限是硬边界，但不能通过随意字符串尾部截断满足。
- 世界书对象包按完整语义行保留，身份、当前事实和当前状态优先。
- 总结超限则拒绝提交并允许重试或调整设置。
- 内部事实不因发布包字数不足而被无规则删除。

默认值只是开箱建议，用户可按模型、世界书容量和玩法修改。

## 10. 事件画像标准

记忆网络同时提供事件画像与对象图谱：事件画像至少显示事件状态、事实数量、来源消息数量、关联对象、总结状态、待结算数量、当前结果和内容规模；对象图谱显示明确关系、关联对象和事件影响。

画像数据从现有仓库计算，不作为事实证据，不触发创建、合并、删除和总结。

## 11. 召回标准

常驻、关键词、向量和现有递归继续承担不同入口。递归只从命中对象受控地带出相关事件或总结，默认保持浅层；不把属性和原子事实扩展成无限图遍历。

焦点只影响 constant，不改变其他排序和容量规则。

## 12. UI 与白盒边界

- 表格用途、表头要求、事实规则、总结规则和字数上限可编辑；
- 固定协议、内部字段键、稳定 ID、事务和聊天隔离不可由提示词接管；
- 事件画像与对象图谱均只读；
- 诊断页可以显示完整 system prompt 和失败阶段；
- 规则恢复应恢复版本化默认值，而不是清空。

# Mirror Abyss 产品契约

## 目标

镜渊不是自动写剧情的导演，也不是每轮重建全部历史的数据库。它负责把游玩中已经发生、可观察、可追溯的事实整理为连续世界状态，并在不阻塞正文的前提下完成沉降与发布。

## 六条边界

1. 世界连续：正文是离散观察，世界状态连续存在。
2. 事实先行：表格、事件和总结只能从已发生事实进入，不从抽象概念直接生成事实。
3. 状态权威：已持久化状态代表当前世界；旧正文变化不反向重算整个世界。
4. 后台分层：审核/修正属于前台，事实提取属于关键后台，总结属于派生后台，世界书属于发布层。
5. 作用域隔离：聊天、消息 revision、任务和世界书事务必须在同一作用域内完成；过期任务只能取消或作废。
6. 概念与执行分离：世界连续、沉降、风声、痕迹、触发精度等概念用于确定模块职责，不直接变成大量硬编码提示词。

## 数据流

`AI 最终正文 → 可验证事实 → 活跃状态表 → 事件小结 → 长期沉降 → 世界书发布`

上游成功可以独立保留。下游失败不得回滚已提交的上游事实；恢复从最早未完成的派生或发布阶段继续。

## 任务优先级

- P400：审核与修正。
- P300：事实提取与状态提交。
- P200：小总结与大总结。
- P100：世界书发布与回读。

同一聊天内保持事实顺序。同一 Profile 采用有界并发；派生总结最多占用一个槽位，避免吞掉关键事实通道。

## 记忆层

- 活跃表：焦点、时空、人物、关系、物品、技能、事件、区域、基础设定。
- 事件小结：发生事实、传播、风声、痕迹、生命周期与触发信息。
- 长期沉降：稳定、不可逆或跨阶段仍有意义的事实。
- 世界书：本地状态的发布视图，不是唯一真相源。

## 历史策略

历史依赖重建已停用。旧消息变化只记录策略事件，不删除当前状态，不调用模型回放旧正文，不阻塞新消息。恢复功能只处理存储、事务、冲突和导出。

# Mirror Abyss rc.43 UI 调度与界面状态修复

## 已复现问题

1. 工作区只订阅 TaskQueue，且任务 queued/running 时仅刷新任务列表并提前返回，阶段卡片长期显示旧的“等待/成功”。
2. 工作区单步按钮未统一检查当前业务链 busy 状态，可能从不同入口重复提交阶段任务。
3. 从历史消息打开工作区后，再通过顶部入口打开，仍保留旧 `selectedMessageIndex`，展示错误消息上下文。
4. 历史恢复存在时同步卡片显示暂停，但同步按钮只检查 `historyInvalidation`，仍可能可点击。
5. 工作区自动重绘可能覆盖正在编辑的对象或表格管理草稿；旧实现若因输入焦点跳过最终刷新，也不会在失焦后补刷新。
6. 任务列表混合其他聊天的历史任务；关闭工作区不会关闭仍打开的对象编辑器。

## 修复

- 工作区同时订阅 pipeline 与 TaskQueue；状态变化统一经微任务合并安排重绘。
- busy 状态综合当前 artifact 的 queued/running 阶段和当前聊天任务队列。
- 业务动作增加统一提交锁；busy 时审核、修正、状态、总结、同步、焦点同步和历史重建入口禁用。
- 输入框聚焦、对象编辑器打开、表格管理草稿存在时延迟自动重绘；安全失焦后补刷新。
- 自动重绘保留主滚动位置、表格横向滚动和 details 展开状态。
- 顶部无 messageIndex 入口显式重置历史消息选择。
- 关闭工作区同步关闭对象编辑器。
- 历史恢复和历史失效任一存在时，同步按钮禁用。
- 最近任务按当前 chatKey 过滤；dispose 时解除 queue 与 pipeline 两类订阅。

## 边界

本次没有调整视觉主题、信息架构、表格字段或业务流程；只修复状态不同步、重复调度和上下文残留。

## 验证

`tests/rc43-ui-scheduling.ts` 与既有 `ui-stage-actions-sync.ts` 覆盖：

- busy 状态建立与解除；
- pipeline/queue 订阅和销毁；
- 运行中不提前跳过 UI 更新；
- 重绘延迟与补刷新；
- 消息选择重置；
- 历史恢复同步禁用；
- 当前聊天任务过滤；
- 旧阶段按钮映射与世界书刷新顺序不变。

# rc.52 固定文本模型协议重构报告

## 问题

旧架构要求模型生成 JSON、遵守 JSON Schema、维护嵌套字段并间接决定对象身份。这同时造成公共反代兼容性差、格式解析脆弱、模型 ID 漂移和同一对象重复创建。

## 新边界

- 模型只填写固定标签文本中的自然语言字段；
- 插件是唯一的数据管理员，负责解析、字段注册表校验、对象匹配、去重、合并、ID 分配和持久化；
- JSON 仅保留为插件内部数据格式；模型输入上下文也转换为固定文本块。

## 删除的旧设施

- 模型 JSON Schema 传输；
- `generateStructuredTask`；
- Schema 能力缓存与结构化 fallback；
- 模型 JSON 语法修复；
- 兼容模式与 JSON 修复设置；
- 审核、状态和总结的旧 JSON 测试夹具。

旧设置键只在加载时被删除，防止历史配置残留。

## 新设施

- 通用固定文本块解析器；
- 状态输入的 `<MA_CONTEXT_FACT>` / `<MA_CONTEXT_INDEX>` / `<MA_CONTEXT_ROW>` 文本序列化；
- 审核固定文本解析器；
- 状态文本协议与插件本地身份分配；
- 总结 slot 路由协议；
- 固定文本连接健康检查；
- 诊断中的 `protocol: fixed-text`；
- rc.52 全链协议回归。

## 防重复机制

1. 同轮按“表格 + 旧 ID 或规范化对象名”合并重复块；
2. 旧对象优先按稳定名称和登记别名匹配；
3. 多个同名对象仅在共同事实、事件、别名或相同内容足以证明时收拢；
4. 存在歧义时停止自动写入，不按数组顺序猜测；
5. 新对象 ID 只由插件生成。

## 验证结论

分段完整历史回归、生产链模拟、类型检查、构建和语法检查通过。安装包不再发布旧 Schema/兼容模式报告；历史文档仅保存在源码包 `docs/history/`。

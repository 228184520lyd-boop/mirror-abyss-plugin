# Mirror Abyss 1.2.0-rc.10 验收记录

## 已自动检查

- TypeScript 严格类型检查通过。
- esbuild 浏览器 ESM 构建通过。
- manifest、运行时版本和设置模板联动通过。
- 当前连接 `generateRaw` 与 Connection Manager 调用路径存在。
- 未包含独立 API 密钥、localStorage/localForage、运行时 loader、生成拦截器或斜杠命令。
- 自适应总结场景通过：密集剧情提前、低变化延后、达到上限触发。
- 历史消息变化会暂停派生处理和世界书发布，并提供手动顺序重算。
- 历史失效期间只禁用相同 `chatKey` 的镜渊世界书条目；重算同步后恢复，不影响共享世界书中的其他聊天。
- SillyTavern 的 `MESSAGE_DELETED` 数值是删除后的聊天长度，不是删除位置；镜渊只使用产物扫描结果，扫描不到时保留现有数据并要求玩家选择重算起点。
- 单次“事实提取与状态表”调用同时产出标准化事实包和完整状态快照。
- 总结优先消费标准化事实；事实也是自适应触发的基础信号。
- 审核若直接给出完整修正版，会复用该正文并复审，不必然再调用修正模型。
- 所有派生结果提交前统一确认当前聊天与正文指纹；不再静默跳过失效结果。
- 小总结保存失败后的回滚也会先确认聊天和正文，切换聊天时不会把旧产物附到新聊天同索引消息。
- 世界书托管条目记录 `chatKey`，同步只更新或删除本聊天自己的条目。
- 模型请求使用轻量 `AbortController`；Connection Profile 可直接取消，当前聊天连接在原生接口不接收 `signal` 时由超时和提交前校验阻止后续写入。
- 任务记录最多保留 200 条，启动上下文最多检查 20 次。
- 插件禁用后解除监听、移除镜渊 UI、拒绝新任务，不清除用户数据。
- 排队和延迟任务记录创建时的聊天归属，切换聊天后不会读取新聊天的同索引消息。
- 请求超时会终止同一个 `AbortController`；支持 `signal` 的 Connection Profile 会取消底层请求，不支持的当前聊天连接允许自然结束但不会继续提交。
- `APP_READY` 回调可在禁用时解除，禁用状态不会被后续事件重新初始化。
- 删除位置无法判断时保留现有派生数据，只暂停同步并要求用户选择重算起点。
- 历史重算区间没有 AI 正文时，会回退发布变更点之前的最新成功快照；若没有任何成功快照，则清除本聊天的镜渊世界书条目，不会留下已删除回合的旧内容。
- 状态提取向酒馆模型接口提供完整 JSON Schema，并明确拒绝缺少九张表的伪成功返回。
- 工作区异步按钮和消息状态条重试按钮提供处理中及失败反馈。
- 主流程失败会同步标记队列任务为失败并交给实际调用入口提示，不再出现“阶段失败但任务显示成功”。
- 审核隔离样式实际隐藏未通过正文、推理块和消息媒体，并保留镜渊处理提示与状态条。
- 关闭消息状态条会立即移除旧面板；重新开启后恢复当前聊天面板。
- 关闭镜渊内部启用开关会立即解除正文隔离和标记；重新开启后按现有审核结果恢复。
- Connection Profile 测试按钮也提供处理中与错误反馈。
- 审核、修正、状态表和总结分别设置适中的原生输出长度；状态表不再沿用过短的聊天默认输出上限。
- Connection Profile 的 Chat Completion 路径会传入原生 `json_schema`，并能接收对象型结构化结果；Text Completion 继续使用提示词与单次格式修复。
- 模型不支持结构化输出并返回空对象时，只额外尝试一次普通 JSON 生成，不形成连续重试。
- 历史编辑、换页和删除事件的异步失败会提示用户，不再只产生未处理 Promise。
- 工作区渲染失败会显示可见错误页，而不是空白或无反馈。
- 插件内启用开关联动请求、队列和排队代次；关闭后旧任务不会在重新启用时复活。
- 启停时失效的排队任务会正常标记失败并释放同 key，重新启用后可以再次提交，不会永久卡在 queued。
- “重建插件缓存（软重置）”会取消请求、失效排队任务、清除一次性世界书模块缓存、解除监听并重新挂载 UI，不删除游戏数据和设置。
- 外部取消会立即结束插件侧等待；即使当前连接的原生请求不接收 `signal`，旧结果也不会继续进入业务提交。
- 定向修正不会吞掉取消或聊天/正文失效错误，禁用或切换聊天后不会继续保存失败阶段。
- 重复触发启用生命周期不会重复安装 `APP_READY` 回调。
- “重置当前游戏”经二次确认后清除当前聊天的新旧镜渊消息产物、聊天元数据和仅属于本聊天的世界书条目，保留其他聊天与插件设置。
- 桌面与窄屏基础布局样式存在。

## 建议在 SillyTavern 中人工验收

1. 安装插件后打开扩展设置，确认镜渊入口和启用开关。
2. 分别测试“当前聊天连接”和一个 Connection Profile。
3. 生成 AI 正文，确认审核、状态表、总结和世界书阶段联动。
4. 编辑旧正文或切换旧 swipe，确认同步暂停及重算按钮。
5. 完成重算后确认世界书恢复发布。
6. 禁用再启用插件，确认没有重复事件或重复 UI。

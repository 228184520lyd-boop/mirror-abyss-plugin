## 自动验证范围

正式 `npm run verify` 包含原 rc.19 回归和 rc.20 专项测试，共 20 个测试文件：

- 人工行、锁定行、遗漏保护行和普通自动行更新；
- 编辑、swipe、删除后的历史修订、活动请求取消、旧任务提交拒绝和聊天隔离；
- 单执行器优先队列、相同 key 去重、生命周期代次、队列裁剪和聊天切换取消；
- 同 lane 单请求、不同 lane 独立、排队取消与请求耗时；
- 自动触发边界、审核/状态 Schema 回退、JSON 修复上限和诊断隐私；
- 事实与状态先提交，派生失败不回滚核心结果；
- 世界书列表、编辑器 DOM 和聊天世界书按钮刷新；
- rc.19 自动链事件合并与历史检查点复用。

## rc.20 专项结果

1. 默认十表初始化及 rc.19 九表迁移通过。
2. 自定义表格新增、字段定义和不定数量注册通过。
3. 删除“活跃事件”视图后内部 `event_id` 仍可总结。
4. 重命名、用途修改、字段修改和顺序调整通过。
5. 状态提示词与 JSON Schema 只包含启用表格。
6. 未注册或停用表格返回被拒绝。
7. 两人对战的纯观众、喝彩事件和观众物品被过滤。
8. 小总结只读取一个 `event_id` 的内部事实。
9. 大总结只读取上一版大总结和未固化小总结。
10. 同一小总结成功固化后不会重复调用模型。
11. `constant / trigger / vector` 三种发布形式通过。
12. `any / all / exclude` 组合触发通过。
13. 世界书按 `fact_id / event_id` 去重及总容量裁剪通过。
14. 历史重建复用审核并使用 `skipDerived` 避免逐条派生请求。
15. rc.19 artifact 事实、旧小总结和旧大总结消费关系迁移通过。
16. 人工行、锁定行和人工世界书条目保护通过。
17. 表格管理新增、删除、保存、启停、排序和恢复默认入口通过。
18. 进行中事件不会因旧阶段已结束而提前沉降。
19. esbuild 浏览器 ESM 构建和最终 `index.js` 语法检查通过。

## rc.21 针对性结果

1. 同一托管键存在多条旧条目时，当前同步只保留一条并删除多余项。
2. 模型更换可见行 ID 但内容不变时，复用旧世界书条目，不再先创建重复项。
3. 没有 `fact_id / event_id` 的完全相同内容在发布前也会去重。
4. 当前场景保持 `constant`，已离开场景切换为 `trigger`。
5. 模型未标注当前场景时，多个时空行中也只允许最后一个有效场景常驻。
6. 单回合大量事实不会触发开放事件小总结。
7. 开放事件达到跨消息边界或明确结束时仍正常总结。
8. 没有合格总结工作时不创建总结任务，UI 阶段显示“跳过”。
9. rc.19、rc.20 原有回归与 rc.21 专项全部通过。

## rc.29 表头结果

1. `title / content / status / keywords` 的默认表头和说明可由玩家覆盖。
2. 覆盖只改变展示和语义描述；`table.key / field.key / type / required / row.id` 保持稳定。
3. 旧快照值、人工基础、完全锁定、事实 ID、事件 ID 和世界书 managedKey 不因改表头迁移。
4. 状态 system prompt 与 JSON Schema description 使用新表头，JSON 属性继续使用稳定键。
5. 表格主列、对象编辑器四个主字段标签和世界书正文采用新表头；默认未修改时维持 rc.28 世界书正文格式。
6. 表头变化使完整状态请求指纹变化；在途旧注册表请求由运行期 fingerprint 守卫拒绝，状态阶段恢复为可重试。
7. 自定义字段纯新增或纯删除保持旧行为；同一次删除旧键并新增新键会被视为危险改键并拒绝，已有类型不可直接改变。
8. 表格定义继续位于全局扩展设置，UI 明示适用于所有聊天；没有新增聊天迁移位。

## rc.28 分阶段结果

1. 阶段一：悬空或非角色焦点清除；合法角色焦点保留；发布集合、排序、容量和正文不受焦点影响。
2. 阶段二：缓存输入覆盖实际完整请求；同名和共享 event/fact ID 不再按排列顺序继承；三层旁观规则一致。
3. 阶段三：所有物理世界书写操作共享一个按最终书名索引的 mutation gate；其他聊天条目不被暂停、删除或改写。
4. 阶段四：普通同步只保留当前聊天稳定身份匹配；历史考古进入玩家主动维护；无变化同步不保存或刷新。
5. 阶段五：当前 schema 绕过历史 artifact 全扫描和深比较；旧 schema 继续执行 V23–V30；没有新增迁移位。
6. 阶段六：五轮生产状态链、五次幂等同步、同步/刷新/再同步、A/B 共享物理书、焦点三态、总结版本与故障隔离通过。

阶段六额外复现并修复：当前数据每次 `putChatState` 都会重跑旧总结消费迁移，导致事实更新后旧小总结消费标记被补回。修复后，兼容迁移只在 V23 事实或 V27 总结版本迁移尚未完成时执行。

## rc.25 针对性结果

1. 39 个 TypeScript 源模块全部包含模块职责和维护边界。
2. 默认注册表仍严格为时空、角色、物品、事件、区域、全局变化、基础设定、自定义对象八类。
3. 旧 `focus/state/skills/relationships` 仅保留迁移兼容，不回到默认视图。
4. 非当前地点仅被提及、没有基础定义或持续变化时，不再残留为区域对象。
5. 具有独立稳定基础定义的区域继续保留。
6. 当前地点设施说明继续只保留在时空。
7. 人工、完全锁定、已固化历史和封锁/损毁/占领等持续区域变化继续保留。
8. `regionAllocationV28` 会对旧聊天幂等重跑区域分配。
9. 每聊天唯一焦点不再影响发布前过滤，设置前后对象集合保持一致，只把对应角色世界书设为 `constant`。
10. 同 ID 的物品或其他对象不会继承焦点常驻。
11. 同一 `event_id` 仍只发布一个 `small:event:<event_id>` 活动小总结条目。
12. 大总结仍只发布 `large:current`。
13. 世界书物理键、逻辑键、完整备注、精确签名和刷新后回读归并链继续通过。
14. rc.19—rc.25 全部既有回归继续通过。
15. 同 ID 的旧 `characters/state` 无论 JSON 键顺序如何，均由角色表提供身份、状态表提供当前内容。
16. `characterMergeV29` 会对已完成 rc.25 迁移的聊天执行一次修复，重复读取保持幂等。
17. `persistedCharacterMergeV30` 即使在 V29 已完成时仍会处理 rc.25 已保存的单表不同 ID 形态。
18. 一条基础行与一条状态行唯一匹配时合并；多个同名候选或判别信号冲突时全部保留。
19. 合并会保留基础/保护层、采用当前状态层并重映射焦点与对象引用。
20. 只缺 V30 时不会全量归一化快照，未注册旧自定义表保持不变。
21. chat 保存失败可完整重试；metadata 保存中断可依靠 artifact ID 别名在刷新后续迁。
22. 并发 `getChatState` 只执行一组保存，普通 metadata 规范化变化也会持久化。
23. 迁移后同步状态改为待同步；旧 generated-ID 世界书条目可在下一次差异同步中退出。
24. 同名不同 ID 角色不会被日常维护按标题无条件删除；稳定 ID 变化会同步改写引用。

## 已知未完成与风险

1. V23–V29 旧全量迁移仍可能在规范化未注册表或未知行字段时丢失未知内容；本版没有重写旧迁移系统。当前 schema 快路径会保留未知表和未知字段。
2. 纯旁观/因果介入为统一领域正则，解决三层不一致，但不是完整语义归因模型。
3. world-info 外部保存一旦开始，插件无法提供跨 SillyTavern 存储层的事务回滚；保存后的提交守卫只保证镜渊状态不会误报成功。
4. 表格定义是全局设置；在一个聊天修改表头会影响所有聊天后续提取和发布，本版只在 UI 明示作用域，没有增加聊天级覆盖。
5. 自动测试无法覆盖真实浏览器 DOM、SillyTavern world-info 缓存、网络时序和第三方扩展交互。

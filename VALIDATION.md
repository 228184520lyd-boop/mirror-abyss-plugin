# Mirror Abyss 1.2.0-rc.13 验收记录

## 自动检查

- TypeScript 严格类型检查通过，并启用未使用变量与参数检查。
- esbuild 浏览器 ESM 构建通过，最终 `index.js` 通过 Node 语法检查。
- 任务优先级测试通过：活动任务完成后，关键任务先于排队中的大总结。
- 任务去重、排队取消、禁用/启用代次和空闲释放测试通过。
- 请求通道测试通过：同一通道按优先级串行，排队请求可取消，不同原生通道互不锁死。
- 触发边界测试通过：只接受 `is_user === false` 的非系统非空正文；玩家、系统、空白和来源不明消息均拒绝。
- 自适应小总结密度策略测试通过。
- 定向修正集成测试通过：`block` 审核仍调用修正与复审，复审通过后原位替换正文和活动 swipe，不新增消息。
- 候选失败隔离测试通过：未提交候选不会覆盖当前正文审核，也不会改动聊天正文。
- manifest、运行时版本和流水线版本一致。
- 运行包不包含独立 API 地址、密钥存储、供应商 `fetch` 客户端、localStorage/localForage 或生成拦截器。

## 静态调用链复核

- 正文关键链和小总结、大总结、世界书发布已拆开，不存在活动任务内部再次等待同一任务队列的嵌套死锁。
- 排队中的大总结优先级低于新正文关键链。
- 同一连接 Profile 仍为单请求通道；没有开放无界并发。
- 插件禁用、聊天切换、历史变化和重置会取消模型请求或排队任务，并通过提交守卫拒绝旧结果。
- 外部 `signal` 与插件控制器已联动，插件主动取消不会再被外部信号覆盖。
- 世界书只更新、暂停或清理同一 `chatKey` 的托管条目。
- 历史事实与状态重算成功后，即使总结或世界书失败，也保留核心结果并显示部分失败提示。
- 诊断数据不保存提示词、正文、模型响应全文或密钥。

## SillyTavern 人工验收项

1. 安装后确认版本显示为 `1.2.0-rc.13`，设置入口与工作区可打开。
2. 分别测试当前聊天连接和一个 Connection Profile。
3. 发送玩家消息，确认不会触发镜渊任务；生成一条 AI/角色正文，确认只触发一次关键链。
4. 在大总结排队时继续生成正文，确认新正文排在尚未开始的大总结之前。
5. 生成途中切换聊天，确认旧任务取消，且新聊天同索引消息没有被写入旧结果。
6. 编辑、删除或 swipe 旧消息，确认世界书暂停并出现重算入口。
7. 重算后确认事实与状态恢复；人为制造总结失败时，核心重算仍保留并给出警告。
8. 禁用再启用插件，确认没有重复事件监听、重复 UI 或旧排队任务复活。
9. 检查诊断报告，确认只有阶段和耗时信息，没有正文、提示词或密钥。

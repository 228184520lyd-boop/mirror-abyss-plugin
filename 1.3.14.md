# Mirror Abyss 1.3.14

流水线：`ma-pipeline-77`

本版把镜渊从“模型逐字段修改表格”推进为“模型输出短自然事实模块，插件统一分发、压缩和结算”。这是状态与总结协议的破坏性升级，旧 `<MA_CHANGE>` 响应夹具不再作为生产兼容输入。

## 主要变化

### 1. 短自然事实模块

每个 `<MA_EVENT>` 只允许一个核心事实。角色、物品、场景、地点、全局和基础设定模块只写对象自身变化，通常一句，复杂时最多两句。多件道具分别成块；模块过长、同对象同层重复、事件已结束却仍含未决事项等情况会被拒绝。

插件只截取模块并确定性分发。模型看不到内部字段、稳定 ID、事实 ID、事件 ID、生命周期和删除操作。世界书不再重复发布核心事件、内部标识和维护权限。

### 2. 总结成为低精度加工层

小总结只统计上一次成功总结后的新增消息，不再因旧来源累计而每回合触发。小总结只写对象近期结果，大总结只写长期结果；两者通过自然文本 `MA_MEMORY` 分发到匹配对象。

精确当前事实被对象结果承接后可退出当前层；大总结完成长期分发后，旧近期文本和处理用事件／事实引用退出。总结正文不是永久世界书副本，消费后可删除，只保留轻量覆盖凭证。

### 3. 自主分发与沉降

明确结束的事件、一次性自动 NPC，以及有明确终态的普通场景和物品，在结果已分发、没有未决事项并被总结覆盖后进入 `settling`，随后从快照和托管世界书真实删除。持续角色、焦点、人工、锁定、重要物品和仍有后续责任的对象保持活跃。

### 4. 基础事实可以被明确事件改变

稳定 ID 不变，但当前基础定义可被明确本质变化事件更新。例如角色通过明确手术从男性变为女性时，当前定义更新为女性，变化经过进入历史；伪装、幻术和临时状态只更新外观或当前状态。完全锁定对象仍不可自动改写。

### 5. 统一 ST 接管

插件启用后默认统一管理托管世界书、向量和递归参数。外围规则审核、向量召回和递归触发可以分别关闭；关闭其中一项不影响其他记忆处理。

### 6. 手机端表格与记忆网络

手机端对象表改为卡片式浏览和编辑，避免长文本把桌面列压成窄长页面。记忆网络移除三层同心圆环，改为焦点／活跃事件中心的关系簇；画布成为主区域，统计与工具条压缩，节点详情固定在图谱下方。

## 验证

本版新增 `rc77-natural-memory-flow`，覆盖自然模块、多道具、核心去重、旧协议拒绝、基础定义变更、总结窗口、对象分发、临时条目物理删除、世界书清理和统一接管。

聚焦命令：

```bash
npm run verify:1.3.14
```

旧聚合 `verify` 包含大量 1.3.13 及更早协议夹具，因此未作为本版一次性通过依据。发布前仍需在真实 SillyTavern 与实际模型上验证自然模块服从度、ST 向量／递归接管和移动端触控布局。

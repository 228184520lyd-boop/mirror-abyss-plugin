# Mirror Abyss rc.58 对象目录与状态上下文报告

## 问题

rc.57 虽未发送完整数据库，但状态请求仍使用最多 120 个逐对象固定文本索引、16 条较完整旧行和 18 条内部事实。对象与事实增长后，固定标签开销和长旧切面会把单次状态提示推高到约 24,000 字符。

## 修复

状态输入拆为两层：

1. `<MA_DIRECTORY>`：按相关性排序、按字符预算覆盖尽可能多的对象；每个对象仅一行，包含表格、稳定名称和最多六个别名。
2. `<MA_CONTEXT_ROW>`：最多八个高相关工作副本，每表最多三个。直接命中对象保留较多当前切面，辅助对象只保留必要的当前信息。

内部事实按正文命中、未决和活跃程度排序，最多选择十二条，并限制每条发生事实、未决事项、关联对象和关键词的上下文长度。

## 不变边界

- 目录不是白名单；新对象仍可自由提取。
- 已有对象是否匹配由插件在完整内部快照中复查，模型不生成 ID。
- 未进入工作副本的已有对象仍可返回本轮变化。
- 未返回对象与字段继续由完整旧快照保留。
- 不限制本轮变化字段数量或独立事实条数。
- 物品既有作用、对象基础定义、近期经历和历史事实的保护规则不变。

## 验证

专项构造八张表、共 240 个长对象。状态用户提示为 10,312 字符；完整工作副本不超过八行。测试同时验证：

- 非相关旧对象仍出现在短目录中，但不携带完整旧行；
- 目录中不存在的新物品可以建立；
- 已有角色发生多字段剧烈变化时沿用旧稳定 ID；
- 固定文本、动态表格、多事件线、历史断点和对象唯一性回归未受影响。

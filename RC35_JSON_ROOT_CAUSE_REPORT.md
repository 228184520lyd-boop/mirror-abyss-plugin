# Mirror Abyss 1.2.0-rc.35 JSON 根因修复报告

## 修复范围

本轮仅处理 JSON、JSON Schema、结构化回退和状态入库前校验，不改变 SillyTavern 的连接调用方式，不重构世界书、总结或历史恢复主链。

## 已确认根因

1. 旧逻辑把普通 400、500 或临时网关故障混入“连接不支持 Schema”结论。
2. 某一份复杂 Schema 被拒后会连带禁用同连接上的其他结构化任务。
3. 状态补丁需要允许省略未变化表格，却使用了容易被误解为字段必须完整出现的严格输出模式。
4. `JSON.parse` 成功后缺少确定性的目标 Schema 校验。
5. 多个 JSON 候选只按“可解析”选择，可能取到解释示例而不是真正结果。
6. 空数组同时承担“无变化”和“清表”语义，存在误清旧数据风险。
7. 截断判断只覆盖对象括号，未完整覆盖数组；状态输出上限仍为 2000。

## 实施修复

- 能力缓存拆为连接级明确不支持与任务/Schema 指纹级拒绝。
- 500/504 等临时故障不写能力缓存；普通回退失败也不写缓存。
- 状态 Schema 使用非严格部分补丁语义，加入 Draft-04、operation/confidence 枚举。
- 新增本地 Schema 子集验证器，不依赖模型二次判断。
- 解析所有候选对象并选择通过目标 Schema 的候选。
- 可解析但结构错误时直接失败；只有纯语法错误才保留一次有限修复。
- 同时检查 `{}` 与 `[]` 是否闭合。
- 空数组变为 no-op；自动清表不再存在。
- 状态 maxTokens 提高到 4096。
- 连接测试返回并显示 Schema 状态。

- 解决上游约 60 秒网关 504。

# rc.47 手机端 UI 修复报告

版本：`1.2.0-rc.47`  
流水线：`ma-pipeline-49`

## 根因

rc.45/rc.46 仅把侧栏改成横向滚动导航，并继续保留 `100vw/100dvh` 与最小宽度 980px 的对象表格。Android 浏览器地址栏、软键盘和安全区变化时会产生高度错误；导航和对象表仍依赖横向滑动，形成实际不可用的手机布局。

## 修复

1. 使用 `visualViewport.height` 同步工作区和对象编辑器高度。
2. 打开工作区时锁定 SillyTavern 底层页面滚动，关闭时恢复。
3. 900px 以下导航改为五列自动换行，620px 以下改为三列，不再依赖横向滑动。
4. 700px 以下对象表格隐藏桌面表头，每一行变成带字段名称的卡片。
5. 处理顶部/底部安全区、全屏对象编辑器、设置入口、按钮组、卡片标题和图谱高度。
6. 380px 以下进一步压缩导航并把表格字段改为单列。

## 未修改

任务调度、状态提取、总结、世界书、对象身份和存储协议均未改动。

## 验证

新增 `tests/rc47-mobile-ui.ts`，并更新 rc.45 布局契约。完整 `npm run verify` 通过。容器 Chromium 无法正常生成本地页面截图，因此最终触控与视觉效果仍需在真实 SillyTavern 手机端验收。

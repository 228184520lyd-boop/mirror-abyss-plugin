# Mirror Abyss 1.2.0-rc.52 业务逻辑审计

流水线：`ma-pipeline-52`

## 本版核心边界

rc.52 将“模型生成结构化数据”和“插件管理内部数据”彻底分离。

模型通信不再使用 JSON。模型输入中的相关事实、旧对象索引和旧行全文由插件序列化为 `<MA_CONTEXT_…>` 固定文本块；模型输出使用任务对应的固定标签文本。模型不再返回 JSON、JSON Schema 对象、稳定 ID、整表快照或数据库补丁。

JSON 只保留为插件内部的设置、快照、任务、表格、迁移和诊断存储格式，不再作为模型通信协议。

## 模型职责

模型只负责读取固定文本上下文，并填写固定文本壳中的自然语言内容：

- 审核：`<MA_AUDIT>`、`<MA_VIOLATION>`、`<MA_REPLACEMENT>`；
- 状态提取：`<MA_TURN>`、`<MA_FACT>`、`<MA_ROW>`；
- 小总结与大总结：`<MA_SUMMARY>`；
- 定向修正：返回完整修正正文，不包数据库结构。

模型不得：

- 生成或选择稳定 ID；
- 直接指定内部行键、事实键或事件键；
- 返回完整 JavaScript/JSON 对象；
- 决定覆盖数据库中的具体记录；
- 复制整张旧表；
- 通过未注册字段扩展内部结构。

固定标签和字段名是协议边界，字段内容允许使用自然语言。缺少必要标签、重复唯一槽位、出现未知槽位或无法闭合的关键结构时，插件拒绝提交，不猜测写入。

## 插件职责

插件是唯一的数据管理员，按以下顺序处理模型输出：

1. 解析固定标签块；
2. 规范化字段名、空白、分隔符和列表；
3. 根据表格、对象名、关键词、别名、事实和事件关联匹配已有对象；
4. 合并同一次响应中的重复条目；
5. 对有充分证据的历史重复行执行收拢；
6. 沿用已有稳定 ID，或由插件为真正的新对象生成 ID；
7. 应用人工锁、基础层保护、字段注册表和历史事务校验；
8. 转换为内部对象并写入聊天快照；
9. 在事务成功后进入总结、世界书同步和诊断链路。

任何模型文本都不得绕过解析器直接写入内部表格。

## 对象身份与去重边界

对象身份不再依赖模型提供的 ID。

优先匹配依据包括：

- 相同表格中的规范化对象名；
- 已登记关键词和别名；
- 已有事实 ID、事件 ID 或明确关联；
- 相同或可证明连续的基础定义与现行内容。

以下情况允许自动合并：

- 同一响应内同表、同对象的重复块；
- 名称或别名唯一命中同一旧对象；
- 多条旧记录具有共同事实、共同事件、共同别名或完全相同核心内容，足以证明是历史重复。

以下情况不得猜测合并：

- 同名但可能是不同人物或对象；
- 一个别名同时指向多条有效记录；
- 仅因同场、围观或内容主题相似而产生弱相关；
- 缺少足够身份依据的跨表对象。

发生歧义时任务失败并保留旧快照，不创建第三条记录，也不静默覆盖。

## 内部 JSON 边界

以下内容继续使用插件内部 JavaScript 对象和 JSON 序列化：

- 扩展设置与动态表格注册表；
- 聊天状态、事实账本和对象快照；
- 小总结、大总结和消费记录；
- 任务队列、请求轨迹和诊断导出；
- 世界书逻辑键、物理键和同步映射；
- 版本迁移与历史恢复检查点。

这些 JSON 全部由插件代码产生、校验和迁移。模型无法直接构造或提交这些对象。

## 已移除的旧模型协议设施

rc.52 运行时不再包含：

- OpenAI `response_format` / `json_schema` 请求；
- 通过 `JSON.stringify` 向模型传递内部事实、索引或旧行；
- strict JSON Schema 模型输出；
- Schema 能力缓存与无 Schema 回退；
- “兼容模式”请求开关；
- 模型 JSON 提取、代码块清理和 JSON 修复重试；
- 由模型生成稳定 ID 或完整表格补丁的路径。

旧设置键仅在加载旧配置时被删除，用于迁移清理，不会重新启用旧协议。

## 人工数据与基础层保护

固定文本协议不改变既有数据保护边界：

- 人工锁定行不得被自动流程删除或覆盖；
- 非空 `baseContent` 原则上不由普通状态提取改写；
- 明确重定义、毁灭、重建或人工编辑才可改变基础定义；
- 未返回的旧行继续保留，空响应不等于清空整表；
- 未注册字段不会进入内部对象；
- 焦点只影响世界书常驻状态，不改变对象身份、排序或裁剪规则。

## 历史重建事务边界

历史失效后，普通状态整理、总结与世界书同步继续受锁约束。

重建过程必须：

- 使用同一聊天范围快照；
- 每次提交前验证聊天、历史修订和消息指纹；
- 失败时保留最后已确认快照和恢复检查点；
- 不把未完成重建的数据发布到世界书；
- 只有重建、发布和必要回读全部成功后才清除历史锁。

固定文本解析失败属于当前消息处理失败，不允许退回旧 JSON 协议，也不允许将原始文本当成有效表格数据提交。

## 世界书同步边界

世界书仍是长期记忆发布层，不是模型输出的直接落点。

同步只消费已经通过插件解析、去重、校验并提交的内部状态。逻辑键、物理键、聊天归属、人工条目保护、保存后回读和重复归并规则继续生效。

## 请求与诊断边界

当前聊天连接和 Connection Profile 均发送普通文本请求，不附带 JSON Schema 或结构化输出字段。

诊断记录协议类型、连接 lane、任务类型、耗时、状态和错误分类，但不记录玩家正文、完整模型输出或 API 密钥。连接成功只证明对应实测请求成功，不以服务对象存在代替真实请求结果。

## 验收结论

rc.52 的模型通信边界已经统一为固定文本协议；插件内部仍保留类型化对象与 JSON 存储。模型负责内容理解，插件负责身份、合并、分发、ID 和事务提交。两层之间不存在模型 JSON、Schema 回退或模型 ID 的旁路。

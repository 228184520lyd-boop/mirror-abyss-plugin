# Mirror Abyss 1.2.0-rc.21 业务逻辑审计

流水线：`ma-pipeline-24`

## 基线与修改范围

本版严格基于上传的 `Mirror-Abyss-1.2.0-rc.19-source.zip` 增量修改。安装包仅用于结构和构建结果复核；未从 GitHub 落后版本重建。保留审核、定向修正、状态提取、小总结、大总结、世界书同步、历史重建、聊天隔离、请求队列、提交守卫、Connection Profile、诊断和既有 UI 主链。

## 数据分层

### 内部事实层

聊天 metadata 中新增独立事实库，核心字段包括：

- `factId`
- `eventId`
- `sourceMessageIds`
- `occurredFacts`
- `unresolvedItems`
- `status`
- `timeRange`
- `relatedEntities`
- `consumedBySmallSummaryId`
- `solidifiedByLargeSummaryId`

事实更新沿用稳定 ID。相同内容重复提交不重开消费；同一 `fact_id` 出现新增经过、未决变化、关闭或替代结果时，清除旧消费标记，使新变化重新进入该事件线的小总结。

### 动态可见视图

固定 `TABLE_KEYS / TABLE_LABELS` 已取消，改为运行时表格注册表。默认十表只承担初始化和“恢复默认”用途，不限制数量。每张表包含稳定键、名称、用途、角色、启用状态、顺序和字段定义。

提示词、JSON Schema、状态返回校验、UI、图谱、沉降和世界书发布均读取注册表。停用或删除表格后：

- 不再要求模型输出；
- 未注册返回会被拒绝；
- 不再出现在状态页和世界书；
- 内部事实、事件线和总结不删除。

## 状态提取边界

状态请求一次产出事实操作和启用表格的完整视图。代码执行以下确定性保护：

- 恢复人工行和锁定行；
- 保留已建立且仍有连续性价值的正式主体；
- 删除焦点与状态表的重复人物卡；
- 过滤纯观众、围观、喝彩、议论、听见、同场及其附属物；
- 两人对战只保留交战双方、对战事实及改变战局的第三方；
- 禁止 snapshot 出现注册表外键。

内部事实会回灌下一次状态提取，用于沿用 `fact_id / event_id`，可见表格不再是唯一数据源。

## 总结链

### 小总结

小总结从未消费内部事实中按 `event_id` 选择一条事件线，只读取该事件事实，不读取全部聊天正文或压缩当前表格。文本保留因果、关键经过、变化、当前结果、未决事项和后续影响。

可见视图沉降不接受模型任意行号，而由代码根据 `event_id / fact_id` 推导，并继续经过人工/锁定、当前相关、未结算和最小保留规则。事件结束判断以该事件线最后一次状态信号为准；旧阶段结束但后续阶段仍活跃或有未决事项时，不退出整条事件视图。

### 大总结

大总结输入固定为上一版大总结和尚未固化的小总结。成功流程为：

1. 生成新累计总结；
2. 写入聊天状态；
3. 回读确认；
4. 标记对应小总结已固化；
5. 标记相关事实已固化。

失败时不修改消费标记；无未固化小总结时不调用模型。

## 世界书发布

托管文档仅使用 `constant / trigger / vector`：

- 常驻：焦点、时空、必要规则、不可缺失状态、直接相关全局变化；
- 触发：`any / all / exclude` 条件被编码为确定性关键词/正则触发；
- 向量：未固化小总结和最新累计大总结。

发布前按模式顺序排列，再按 `fact_id / event_id` 去重并按字符总容量裁剪。旧 `vectorizeRows` 仅保留设置兼容，不能把总结降级成无触发的普通条目。`order` 只用于编辑器显示，不参与记忆取舍。

SillyTavern 原生 World Info 只支持条目级 `vectorized` 标记；实际分数门槛和 Max Entries 属于 Vector Storage 全局设置。本版不修改用户全局配置，镜渊侧数值只用于托管元数据和发布候选裁剪。

## rc.21 世界书与总结调度修复

### 世界书差异同步

同步不再把托管键映射成单值。当前聊天的所有镜渊托管条目都会进入候选集合：

- 优先按稳定托管键复用；
- 行 ID 变化时按规范化内容复用；
- 同键、同内容或旧行 ID 留下的未认领托管条目统一删除；
- 人工条目和其他聊天条目不进入候选集合。

发布文档层增加无 ID 的精确内容去重，避免模型重复输出相同视图时生成两个条目。

### 场景常驻切换

`spacetime` 不再整表强制常驻。只有一个当前有效场景可为 `constant`；状态包含“已离开、历史场景、已结束、归档”等信号的条目转为 `trigger`。模型未明确标注时，以最后一个非历史时空行为当前场景。状态提示词同时要求场景切换时更新旧场景状态和触发词。

### 总结调度

自动派生前先根据内部事实和未固化小总结计算工作计划。无合格工作时总结阶段直接标记 `skipped`，不创建模型任务。开放事件取消“单回合事实数量”触发条件，只在事件明确结束或达到跨消息边界时自动生成小总结。

## 历史重建与提交边界

rc.19 的检查点复用和提交守卫保持：

- 变更正文才重置该条审核/修正；
- 后续未变正文复用审核结果，仅重建依赖上游的状态；
- 核心状态重建时跳过派生，避免每条消息都触发总结/世界书；
- 重建完成后按符合条件的受影响事件线排空小总结，再只固化未固化小总结；
- 世界书最终重新发布，不调用模型；
- 旧任务仍受生命周期代次、历史修订号、`chatKey`、消息身份和正文指纹守卫。

## 迁移与保护

- 旧九表映射到新默认视图，并新增“全局变化”；
- 旧人工行和锁定行立即落盘保存；
- rc.19 artifact 事实包迁入内部事实层；
- 旧小总结 `sourceKeys` 和旧大总结消费关系转换为显式消费/固化标记；
- 世界书同步只删除或更新同时满足镜渊托管标记与相同 `chatKey` 的条目；
- 人工世界书条目不删除、不禁用、不改写。

## 明确保留的边界

- 不生成或拦截 SillyTavern 主叙事正文。
- 不保存独立 API 地址、密钥或供应商配置。
- 不切换 SillyTavern 全局 Connection Profile。
- 不修改 Vector Storage 全局门槛或 Max Entries。
- 不引入业务并发；保留任务队列和同 lane 单请求调度。
- 不做与动态记忆结构无关的架构重写。

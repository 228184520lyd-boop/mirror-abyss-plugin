# Mirror Abyss 1.2.0-rc.23 业务逻辑审计

流水线：`ma-pipeline-26`

## 基线与修改范围

rc.23 以正式 rc.22 源码为直接基线，继续保留审核、定向修正、状态提取、小总结、大总结、世界书同步、历史重建、聊天隔离、任务队列、提交守卫、Connection Profile、诊断和自动流程。本版只完成对象视图与单一焦点常驻阶段，不提前实现总结影响分发和长期事实固化重构。

## 对象化可见视图

默认视图由 rc.22 十表收拢为八类：时空、角色、物品、事件、区域、全局变化、基础设定、自定义对象。

表格仍是动态视图，不是唯一数据源。删除或停用视图不会删除：

- 聊天级内部事实；
- `fact_id / event_id`；
- 小总结和大总结；
- 历史重建依据；
- 玩家人工世界书条目。

### 角色对象

关系和技能不再默认单列，而是进入角色对象：

- `baseContent`
- `solidifiedHistory`
- `currentStates`
- `relationshipStates`
- `abilityStates`
- `relatedObjects`
- `relatedEvents`

关系状态必须由明确关系变化支持；能力状态必须由明确使用、获得、失去、受限或改变支持。同场、普通对话、表情、推测或身份联想不能自动赋予。

### 其他对象

物品、区域、事件、基础设定、全局变化和自定义对象使用同样的基础层、已固化历史和当前状态层。玩家新增普通对象默认基础保护；完全锁定对象保持全字段保护。

## 基础层写入边界

状态提取只允许：

- 对新对象写入正文首次明确的稳定基础；
- 更新当前状态、关系状态、能力状态和关联；
- 保留已有基础内容与已固化历史。

一旦 `baseContent` 非空，后续状态模型不能改写。`solidifiedHistory` 不属于状态提取写区；旧值原样保留，模型新增值被丢弃。

## 旧数据迁移

### 无歧义映射

- `state / characters → characters`
- `events → events`
- `regions → regions`
- `foundations → foundations`
- 其他同名对象视图直接沿用

### 关系与技能归并

旧关系根据角色稳定名称归并：

- 明确命中一个或两个角色时写入对应 `relationshipStates`；
- 命中三个以上、没有命中或存在歧义时转入自定义对象并标记“待归并”；
- 不允许任取前两个角色。

旧技能只有唯一角色归属时写入 `abilityStates`，否则转为待归并。

人工或锁定旧关系/技能成功归并后，原记录保留为“已归并”迁移凭据，继续受人工保护，但不会再次发布世界书，因此不产生内容副本。

自动焦点行不属于事实，不迁移。人工或锁定焦点备注保留为自定义对象。

## 单一焦点常驻

每个聊天在 `ChatState.focusObjectId` 中最多保存一个玩家手动选择的角色。

焦点仅产生一项业务效果：

```text
该角色世界书条目 recallMode = constant
```

明确不产生：

- 额外排序；
- 权重；
- 容量锁定；
- 裁剪豁免；
- 召回优先分；
- 正文中的“玩家控制”描述；
- 模型自动设置或切换。

焦点判断同时校验表格角色，只对 `characters / state` 生效。其他表出现相同 row ID 也不会常驻。玩家切换焦点时只更新当前聊天 metadata 并强制重新同步世界书，不调用模型。

## 世界书边界

rc.22 的精确重复归并继续保留。对象文档稳定键仍为：

```text
view:<tableKey>:<rowId>
```

焦点只改变对应角色文档的 `constant`。其他世界书模式继续由既有业务规则和 SillyTavern 触发/向量机制处理。

“已归并”的人工迁移凭据只保存在镜渊 UI 和聊天数据中，不发布为世界书条目；普通未归并人工对象仍正常发布并受保护。

## 总结与事实层保持

内部事实层、小总结 event_id 汇总、大总结增量消费、固化标记和历史重建逻辑保持 rc.22 行为。

本版没有实现：

- 小总结向多个对象状态分发；
- 大总结向对象已固化历史分发；
- 多事件影响的独立 add/update/close/replace 操作层；
- 多事件状态综合与冲突队列。

这些能力将在对象视图完成实机确认后作为后续独立阶段开发，避免和基础迁移同时引入大范围不可验证变化。

## 历史重建与提交边界

- 正文和审核规则未变化时复用审核；
- 只从受影响状态开始重建；
- 核心重建期间跳过逐条派生；
- 小总结只重建受影响 event_id；
- 大总结只消费未固化小总结；
- 世界书重新发布不调用模型；
- 所有写入继续受任务代次、历史修订号、chatKey、消息身份和正文指纹守卫。

## 明确保留的边界

- 不生成或拦截 SillyTavern 主叙事正文。
- 不保存独立 API 地址、密钥或供应商配置。
- 不切换 SillyTavern 全局 Connection Profile。
- 不修改 Vector Storage 全局门槛或 Max Entries。
- 不引入业务并发。
- 不删除人工世界书条目。
- 不把表格重新提升为事实真相来源。

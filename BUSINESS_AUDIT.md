# Mirror Abyss 1.2.0-rc.18 业务逻辑审计

## 产品主链保持不变

1. 明确的角色正文进入规则审核。
2. 审核通过，或定向修正并复审通过后，正文进入记忆链。
3. 单次状态任务同时产出标准化事实包与九类当前状态快照。
4. 事实与状态先保存到对应消息和当前聊天元数据。
5. 小总结消费尚未总结的事实，并执行代码约束下的安全沉降。
6. 大总结合并尚未消费的小总结和上一版长期总结。
7. 最新状态、总结和连续性文档发布到当前聊天世界书。
8. 历史变化后暂停派生发布，由用户选择起点顺序重算。

## 本版修复的已确认问题

### 生命周期与提交

- 任务队列增加运行代次守卫。禁用、重启或重置后，即使底层供应商忽略取消信号，旧任务也不能继续提交。
- artifact 提交守卫绑定任务代次、历史修订号、`chatKey`、消息身份和正文指纹。
- 聊天切换会取消旧聊天尚未开始的排队任务；活动请求继续使用既有取消与提交拒绝边界。

### 调度与派生链

- 保留单执行器，不开放业务并发。
- 活动任务不抢占；待执行任务按优先级选取。正文关键链为 90，世界书为 40，小总结为 30，大总结为 10，人工操作为 70。
- 状态成功后立即提交核心事实与状态，自动总结和世界书不再留在同一个正文任务内等待。
- 新正文核心状态提交时会取消同聊天尚未开始的旧自动派生任务，避免较旧快照随后发布。
- 小总结、大总结和世界书分别处理失败；一个派生阶段失败不会抹掉核心结果，也不会无条件阻止后续可执行阶段。

### 模型连接

- 当前聊天连接使用 `current-chat` lane；Connection Profile 使用 `profile:<id>` lane。
- 同一 lane 同时一个请求，不同 lane 可独立运行。
- lane 内按审核/修正、状态、小总结、大总结排序；不抢占已经运行的请求。
- 记录 `transportWaitMs`、`requestMs`、`totalMs`，流式模式仍固定为关闭。

### 触发、修正与诊断

- 自动触发只接受明确角色正文；不再把系统消息、来源不明事件或无法解析的事件退回到聊天最后一条消息。
- 修正连接、超时或解析故障不再被解释为内容修正失败，也不会因此套用隐藏正文的内容回退。
- 审核提示词和 JSON Schema 对 `replacementText` 的要求一致；字段为空时会调用修正模型，而不是把空字符串当作完整修正版。
- 审核结构化请求若因供应商不支持 Schema、返回 `{}` 或出现内部服务器错误，只执行一次无 Schema 兼容回退；两条路径都失败时保留技术失败，不隐藏正文。
- 状态提取使用同样的单次 Schema 兼容回退；该回退不受“JSON格式修复”开关影响，回退文本仍可使用既有的一次格式修复。
- 当前连接和 OpenAI Connection Profile 的连接测试也会在 Schema 请求报错或返回 `{}` 时只回退一次。
- 审核解析不再把缺少 `result` 的空对象当作 `revise`；格式修复目标结构已补齐 `replacementText`。
- 复制诊断只包含数量、状态、耗时和错误，不包含玩家输入、角色正文、小/大总结正文、完整模型响应或密钥。
- 任务记录裁剪只删除已结束历史，不隐藏正在运行或排队中的任务。
- 任务状态增加 `cancelled`，并记录队列等待、运行和总耗时。

### 分阶段 UI 与世界书刷新

- 最新角色消息下方新增审核、重新修正、表格、小总结、大总结、世界书同步六个独立操作按钮；工作区总览、审核页、表格页和总结页保留对应入口。
- `retryStage` 的审核/修正不再调用完整主链，符合“单一阶段失败只重试该阶段”的 UI 文案。
- 单独修正成功后清理不再对应当前正文的旧状态和派生总结，并暂停旧世界书条目，等待新表格。
- 最新 AI 消息自身发生编辑或 swipe 时，显式完整整理成功后可解除该条历史失效；不放宽更早历史和未知删除保护。
- 状态表提交后立即发布派生阶段状态，避免 UI 与后台队列脱节。
- 保存世界书后先刷新 `world_names` 列表，再重载目标编辑器；目标书当前显示或强制同步时继续等待 `showWorldEditor` 完成 DOM 重绘。
- 手动同步在内容无变化时也可刷新目标世界书；新建并关联聊天世界书后立即更新酒馆聊天世界书按钮状态。

## 明确保留的边界

- 不生成或拦截主叙事正文。
- 不保存独立 API 地址、密钥或供应商配置。
- 不切换 SillyTavern 全局 Connection Profile。
- 不自动删除酒馆消息。
- 不增加事务数据库、outbox、工作流引擎或跨聊天协调器。
- 不开放无界并发，不强杀已经运行的模型请求。
- JSON 结构修复仍最多一次，不形成递归调用。
- 九类表、事实包、总结对象和 ChatState 数据格式未改变。

## 验证边界

离线回归、构建、语法检查和依赖审计已经完成。真实 SillyTavern 事件负载、真实供应商取消行为、世界书 API、长时间多聊天压力和移动端 UI 仍需实机验收。

# rc.53 对象唯一性与世界书归并报告

## 核心规则

同一聊天、同一表格、同一规范名称只能存在一个对象。

事实、事件、时间和状态差异是同一对象的不同切面，必须写入 `currentFacts`、`currentStates`、`recentHistory`、`solidifiedHistory`、事实 ID 与事件 ID 等层，不得通过创建同名行表达。

真正需要区分的同名对象，必须在首次建立时使用不同的规范名称；世界书出口不再使用 ID 后缀临时保留同名条目。

## 修复范围

- 新增全链统一的对象名称规范化函数；
- 状态文本解析、快照归一化、保护层合并、手工编辑、历史迁移和世界书发布共用同一名称身份；
- 同轮重复 `<MA_ROW>` 在插件内合并；
- 同一快照中的同表同名行无条件合并；
- 不同历史快照中的同一规范对象沿用首次稳定 ID；
- 合并时汇总关键词、事实 ID、事件 ID、数组字段与召回条件；
- 人工基础保护和完全锁定优先级保持不变；
- 关联对象引用、玩家焦点 ID 与旧 ID 映射同步改写；
- UI 手工新增或改名命中同名对象时直接更新原对象，不再 `push` 新行；
- 世界书待发布计划按完整托管名称执行最终唯一化；
- 当前聊天的同名托管副本确定性保留一个 UID，删除其余副本；
- 镜渊自动创建的专用世界书可接管并清理旧版无归属 `[MA11]` 副本；共享世界书中其他聊天与人工条目不受影响。

## 一次性迁移 V31

升级后首次读取旧聊天时执行 `uniqueObjectNamesV31`：

1. 扫描全部消息快照的全部注册表格；
2. 合并同一快照中的同名对象；
3. 统一跨快照稳定 ID；
4. 改写 `relatedObjects` 与 `focusObjectId`；
5. 将旧 ID 映射保存到消息 artifact，支持聊天已保存但 metadata 保存失败后的安全续迁；
6. 清除旧世界书物理 ID 缓存并将同步状态恢复为 `idle`；
7. 下一次同步按唯一对象视图重建并清理同名副本。

迁移幂等：完成位写入后不重复扫描；保存失败时按事务边界回滚或通过持久化别名续跑。

- 跨快照稳定 ID 继承；
- 关联引用与焦点 ID 改写；
- 同步缓存失效；

- 迁移完成后的幂等读取。

# Mirror Abyss 1.3.11

本版不增加新的记忆功能，目标是删除重复权威，使同一条目只由一套状态机改变。

## 运行内核

```text
明确事实
→ 稳定对象匹配
→ 特殊表规则
→ 生命周期迁移
→ 保存提交态
→ 总结覆盖
→ 覆盖审计与物理删除
→ 世界书只读发布
```

- `memory-state-machine.ts` 统一编排生产记忆变化。
- `entry-lifecycle.ts` 是参与状态和删除的唯一执行器。
- `snapshot.ts` 只应用已确定补丁并提供只读工具。
- 小总结／大总结只写覆盖结果，不直接删除条目。
- 世界书、事件画像、关系图谱和 UI 只读取提交态。

## 删除与隔离

- 删除重复的旧沉降模块 `sedimentation.ts`。
- 删除无生产入口的 `summary-policy.ts`。
- 模型返回的旧生命周期字段不再被执行。
- `absorbed / retired` 只在旧存档加载迁移时处理，正常主链仅允许 `active / settling`。

## 兼容性

旧存档会在首次加载时执行 schema 3／memory state 2 的一次性迁移。缺少事实凭证的旧墓碑不会被猜测删除；迁移完成后不会每轮重复运行旧兼容逻辑。

## 验证

新增 rc74 单一权威测试，并通过 rc68–rc74、历史恢复、对象唯一化、总结事务、世界书、记忆网络、TypeScript、构建与语法检查。聚合测试受外部总时限中止，全部组成命令已分别通过。

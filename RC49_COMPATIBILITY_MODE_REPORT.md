# rc.49 兼容模式

## 目标

只增加一个用户可理解的兼容开关，不引入自动切换连接、错误分类路由或多级降级。

## 行为

- 关闭：沿用 rc.48 的 JSON Schema 请求与既有有限回退。
- 开启：结构化任务从第一次请求起不携带 `jsonSchema` / `response_format`。
- 返回内容仍由镜渊执行 JSON 提取、本地业务 Schema 校验和可选的一次语法修复。
- 不切换 Connection Profile，不改用 current-chat，不自动重复网络请求。

## 覆盖范围

规则审核、状态提取、小总结、大总结以及连接测试。定向修正原本不是 Schema 请求，不受影响。

# Mirror Abyss 1.1.0-rc.4.2 验证记录

## 修改范围

- 补回事实包索引 `byId`。
- 小总结预算 `12,000 → 5,000` 字符。
- 小总结默认阈值 `15 → 7` 回合。
- 大总结默认阈值 `6 → 4` 个小总结。
- 对精确旧默认值 `15/6` 执行一次设置迁移。
- 在小总结输入顶部列出本批唯一允许的事实 ID 与变化行 ID。
- 严格来源校验保持不变。

## 自动检查

- 八个精确补丁全部单次命中：通过。
- 已修复输入再次执行保持幂等：通过。
- 重复或歧义标记拒绝启动：通过。
- `core-hotfix.js` ES Module 语法：通过。
- `finalizer.js` ES Module 语法：通过。
- JSON 元数据解析：通过。
- 覆盖包不包含 `index.js` 或 `style.css`：通过。
- 设置迁移仅处理旧默认 `15/6`，其他用户值保留：静态检查通过。
- 保存的旧批次会在恢复时按当前预算重新构造并拆分：现有代码路径检查通过。

## 实机验收

仍需在 SillyTavern 中执行真实小总结。重点观察：

- 单批显示的预算占用不超过 5,000。
- 模型输出的来源 ID 均来自顶部白名单。
- 失败后重试不会重复提交总结。
- 累计 4 个小总结后大总结正常触发。

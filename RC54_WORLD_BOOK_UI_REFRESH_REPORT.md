# rc.54 世界书界面刷新修复

## 现象

“按对象清理并重新发布”已经把重复条目从缓存和服务端删除，但当前世界书编辑器仍显示旧列表；刷新网页后重复项消失。

## 根因

旧刷新函数在目标书已打开时同时执行：

1. `reloadEditor(name, true)`：触发选择框 `change`，异步刷新；
2. `showWorldEditor(name)`：立即执行实际重绘。

两条刷新链并发，延迟完成的旧 `change` 可能再次把旧 DOM 覆盖到编辑器中。数据实际已经正确，因此整页刷新后显示正常。

## 修复

- 当前目标书：只调用一次可等待的 `showWorldEditor(name)`。
- 等待两帧 DOM 提交后再进行保存后确认。
- 不再同时触发 `reloadEditor`。
- 后台未打开的世界书：仍只调用 `reloadEditor`，不打断玩家当前编辑器。

## 边界

本次不修改对象合并、稳定 ID、世界书发布内容或旧数据迁移规则，只修复保存成功后的 UI 刷新竞态。

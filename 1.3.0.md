# Mirror Abyss 1.3.0 — 标准工作台

发布日期：2026-07-20  
流水线：`ma-pipeline-62`

## 本版目标

把已有的长期记忆能力整理成开箱可用标准：默认提示词可以直接运行，用户需要修改的判断全部白盒化，普通界面不再暴露数据库协议，同时保留诊断和高级可追溯性。

## 主要变化

### 规则中心

- 对象准入从两块扩展为四块：允许建档、默认排除、证据与不确定性、更新与冲突。
- 小总结和大总结五段规则统一移入“规则”页。
- 提供标准/已自定义状态、恢复标准规则和完整 system prompt 诊断预览。
- 固定文本协议不再出现在普通设置流，只在诊断页只读展示。

### 表格

- 默认角色、物品、事件、区域、全局变化、基础设定和自定义表用途全部收紧。
- 对象页增加搜索、结果计数和关键列固定，移动端保持真实列结构。
- 表头仍采用“表头名称｜记录要求”，内部键和类型保持隐藏。

### 图谱

- 优先读取关系状态、关联对象和关联事件。
- 旧文本提及只在缺少显式关联时回退，并使用独立样式。
- 增加搜索、高亮、节点连接度、关系统计、图例和缩放。
- 图谱保持只读，不反向成为事实来源。

### UI 与引导

- 工作区、设置区和正文处理区移除折叠面板。
- 总览新增六项开箱检查与动态进度条。
- 功能按总览、对象、规则、图谱、总结、审核、发布、设置、诊断分配。
- 普通页面减少 constant、trigger、vector、稳定 ID 等底层术语。
- 增加实时状态区、搜索反馈、复制提示和尊重 reduced-motion 的轻量动效。

## 数据与兼容性

- 无破坏性数据结构变更。
- 旧版两段对象准入设置会自动补充证据与更新标准默认值。
- 旧图谱记录仍可通过兼容回退显示。
- 稳定 ID、匹配、去重、事务提交、历史恢复、聊天隔离和世界书安全边界未被提示词接管。

## 验证

新增 `npm run test:rc62:standard`，覆盖四段状态规则、标准工作台、非折叠布局、开箱进度、表格搜索、图谱显式关系优先、兼容回退和动态效果边界。

同时通过动态表格、可编辑表头、固定文本协议、唯一对象、快照修订、对象目录、聊天切换、异常响应、分离保存、玩家运行流程、TypeScript、构建和语法检查。

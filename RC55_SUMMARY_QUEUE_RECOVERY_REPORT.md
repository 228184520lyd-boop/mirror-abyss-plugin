# rc.55 小总结空响应与队列恢复报告

## 问题

Connection Profile 返回空内容时，小总结正确报出 `No message generated`，但旧的持久化 `summary=queued/running` 可能继续被 UI 当作互斥锁；玩家再次点击立即小总结时，陈旧自动派生任务也可能排在手动请求之前，形成长期“在队列中”的外观。

## 修复

- 空响应失败后验证任务 `inFlight` key 必须释放。
- 手动总结取消同聊天尚未执行的自动小总结、大总结和同步任务。
- 手动总结请求取消正在运行的旧自动总结，并沿原单执行器安全收尾。
- 手动总结任务记录 `messageKey`、指纹、历史修订和触发来源。
- 工作区和消息面板只在真实队列存在对应任务时，将总结/同步视为忙状态。
- 发现没有真实任务支撑的陈旧总结状态时，手动入口先把它转为可重试失败状态并持久化。

## 不变边界

- 不对空响应自动伪造内容。
- 不增加模型自动重试次数。
- 不改变固定文本协议、事件线总结算法、事实消费、世界书结构和对象身份。

## 验证

任务队列、生成器 lane、UI 阶段动作、自动派生、部分失败、历史恢复、恢复竞态、固定文本协议、世界书刷新、类型检查、构建和语法检查均通过。

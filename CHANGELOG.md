# 更新记录

本文件只记录对用户和维护者有意义的正式版本变化。开发候选版本的详细报告保存在 `docs/history/`。

## 1.3.6 — 2026-07-21

### 无观点事实书记

- 状态与总结提示词收紧为“无观点事实提取／压缩”：禁止模型评价重要性、预测未来、解释动机、识别伏笔、补全未知、判断结算或提出条目合并与删除。
- 生产状态链忽略模型返回的旧生命周期指令；模型只返回本轮明确事实及其对象字段投影，稳定 ID、去重和生命周期继续由插件确定性执行。
- 旧 `unresolved` 输出不再进入新事实和总结；既有历史字段继续只读兼容，避免升级破坏旧聊天。

### 增量整理与事件画像

- 新增插件侧增量结算器，只检查本轮触及条目和同事件明确终态：普通可替代物品、已结束场景与已结束事件可进入待结算；核心、唯一、任务、证物等物品受固定规则保护。
- 日常游玩不把完整仓库正文交给模型。状态上下文只提供相关候选短目录与少量高相关工作副本，降低输入增长和 504 风险。
- 默认工作区“关系图谱”改为“事件画像”。画像由事实、对象快照、小总结、大总结和待结算状态实时派生，不保存第二份正文、不新增模型调用。
- 当前关键词、向量、常驻和现有递归行为保持不变。

### 属性分层与白盒硬上限

- 角色表增加最小属性槽位：身份与对象定义、长期与现行事实、当前状态、外观与表现。身份锚点只接受正文明确纠正、转化、毁灭、重建或人工编辑；外观和单次情绪不得改写身份。
- 新增每种对象表、小总结和大总结的白盒硬字数上限。世界书发布按字段优先级保留完整事实行；总结超过上限直接拒绝提交并进入可重试失败，不进行字符串尾部硬截断。
- 默认上限随版本迁移补齐，不覆盖用户之后的自定义值。

### 验证

- 新增 `rc69-fact-only-incremental`，覆盖无观点提示词、旧未决输出屏蔽、模型生命周期指令忽略、插件增量结算、唯一物品保护、事件画像、角色属性分层和世界书单条硬上限。
- 全部历史回归组成命令、六次 3.0–3.4 秒玩家流程、TypeScript、构建和语法检查均通过；一次性聚合命令受外部 900 秒上限中断，不将该次超时声明为退出码 0。

## 1.3.5 — 2026-07-21

### 世界书参与状态与结算

- `absorb / retire` 改为短期 `settling`：原条目仍是同一世界书对象，但同步为 `disable=true`，不参与常驻、关键词、向量或递归召回。
- `absorb` 先把有效后果、事实/事件引用和必要名称分发到唯一宿主；`retire` 只接受明确终态且无宿主后果的对象。
- 小总结或大总结完成后执行覆盖审计：事件已结束、相关事实已被总结覆盖、宿主已收到分发结果且对象无保护时，物理删除原表格行和托管世界书条目。
- 支持级联结算：临时角色或物品可并入事件，事件容器自身也可在同一总结覆盖后删除，不因宿主同时待结算而留下永久中间节点。
- 待结算阶段可显式 `restore`；人工、锁定、基础保护和焦点对象继续禁止自动退出。

### 旧数据迁移

- 1.3.4 的 `absorbed / retired` 隐藏墓碑只作为兼容输入。其事实已进入小总结/大总结、没有活跃或未决事实且宿主承接完整时，后续状态维护会直接清除。
- 新版本不再生成永久隐藏墓碑；插件仅保留完成世界书停用、总结分发和事务提交所需的短期结算状态。

### 验证

- 扩展 `rc68-entry-lifecycle-absorption`：覆盖世界书停用、事件未结束时保留、总结覆盖后真实删除、待结算恢复、保护条目、无宿主退出、级联删除及 1.3.4 旧墓碑回收。
- 全部历史回归组成测试、3–3.4 秒玩家流程、TypeScript、构建和语法检查均通过；一次性聚合命令受 300 秒环境上限中止，未将该次超时声明为退出码 0。

## 1.3.4 — 2026-07-21

### 条目生命周期

- 新增 `absorb / retire / restore` 生命周期指令。自动条目失去独立价值时可并入宿主、无宿主且完全失效时可退出、重新获得独立价值时可恢复。
- 并入不会物理删除源对象。源对象保留为隐藏墓碑，不参与普通表格、图谱、条目计数和世界书发布，用于阻止相同对象被反复新建。
- 宿主通过插件专属“归并记录”接收有效后果，并继承源条目的事实引用、事件引用、名称和关键词召回。
- 世界书下一次同步会移除源对象的独立托管条目，同时让旧名称触发宿主条目。

### 安全边界

- 人工、锁定、基础保护和焦点条目禁止自动退出。
- `absorb` 必须指向唯一存在的有效宿主并提供归并事实；同一对象不能并入自身。
- `retire` 只接受已出售、赠出、消耗、损毁、丢弃、结束、解决、解散或明确无后续价值等终态。
- 普通状态更新不能让隐藏墓碑重新占位；只有显式 `restore` 才能恢复独立条目。
- 不改变审核、修正、总结、历史恢复、对象稳定 ID 和世界书事务提交顺序。

### 验证

- 新增 `rc68-entry-lifecycle-absorption` 回归，覆盖出售物品并入交易事件、宿主召回继承、世界书源条目退出、墓碑防重建、显式恢复、锁定保护和无宿主退休。

## 1.3.3 — 2026-07-21

### 物品覆盖

- 将物品准入从“必须长期变化”改为“只要可区分且会影响下一轮连续性”：所有权、位置、数量、完整性、可用性、隐藏状态、证据/任务用途、独特功能或后续能否取得任一项成立即可建档。
- 状态提示增加逐件扫描要求，不得只记录最显眼物品，也不得把多个独立命名物品合并为一条。
- 状态输出预算从 4096 提升到 6144 token，降低多物品、多对象同轮返回被截断的概率。
- 世界书发布保留没有角色关联但自身重要的物品，不再把“未被角色持有”误判为无效。

### 场景条目

- 默认注册表新增“场景”视图，区分“本次局面切片”和“可复用地点本体”。
- 场景记录参与对象、核心局面、直接限制、环境状态和未决承接；场景结束后归档为已结束/已离开，不删除。
- 旧设置升级时在“时空”后自动补入场景表，不覆盖玩家已有表格或白盒规则。
- 当前时空存在但模型漏写场景时，插件从当前时空生成最小场景条目，并沿用确定性的稳定 ID。

### 语义归档纠正

- 每个 `<MA_ROW>` 增加 `kind`：`spacetime / scene / character / item / event / region / global / foundation / custom`。
- `kind` 由对象本质与作用域决定，插件用它校验模型选择的 table；组织、阵营、政权、机构、群体格局和制度执行即使具名并会行动，也归入全局变化。
- 旧自动条目被确认误归档时，保存前从源表迁移到目标表，保留稳定 ID、基础定义和通用字段，并移除目标表不支持的角色专属字段。
- 人工、基础保护或完全锁定条目不自动迁移，继续由玩家明确处理。

### 兼容与边界

- 不改变审核、定向修正、小总结、大总结和世界书事务流程。
- 标准状态 system prompt 经去重后控制在 5,200 字符以内。

## 1.3.2 — 2026-07-21

### 条目类型分流

- 将默认“区域”视图改名为“地点”，恢复重要地点、建筑、房间、道路与秘境的独立条目入口。
- 状态规则新增可编辑的“类型与表格分流”，以对象本质和事实作用域为准，而不是根据名称、行动或拟人化表述分类。
- 角色表只接收具体个体人物、独立人格生物或意识体；组织、阵营、政权、机构、群体、地点、制度与世界规则禁止回退到角色表。
- 全局变化明确接收组织、阵营、政权、机构、群体格局、制度执行与世界态势，即使其叙述形式类似角色。
- 已有对象发生模型误选表格时，按跨表唯一名称锚定原归属，避免同一对象在角色与全局表之间跳动。

### 人工修正

- 条目编辑器新增“归属表格”选择，可将既有误分类条目移动到地点、全局变化或其他启用表格。
- 移动时保留稳定 ID、事实引用和人工保护语义；目标表不支持的角色生命周期字段会安全移除。

### 审核恢复

- 将同一正文、同一规则下已经完成的失败审核视为正式检查点；历史恢复从该消息继续时直接进入定向修正，不再重新审核后把修正误标为“跳过”。
- 玩家手动点击“重新审核”仍会强制发起新审核，不受检查点复用影响。
- 未提交的候选修正文审核只保存在修正记录中，不再覆盖当前可见正文对应的审核结果。

### 边界

- 不自动猜测并批量迁移历史误分类条目，避免把同名人物、组织和地点错误合并；由玩家在编辑器中明确修正。
- 状态系统提示在新增分流规则后仍控制在 4,700 字符以内。

## 1.3.1 — 2026-07-20

### 移动端 UI 修复

- 修复设置页复选框被通用输入框样式拉伸，导致蓝色横条和文字逐字竖排的问题。
- 手机横向查看表格时取消固定前两列和模糊覆盖层，避免滑动内容被“玻璃层”遮挡。
- 放大移动端标题栏与触控区域，并将导航顺序调整为“总览—表格—规则—总结—图谱—审核—世界书—设置—诊断”。
- 在表格、规则、总结和设置页明确显示“状态表 → 小总结 → 大总结”的写入关系。
- 聊天正文下方恢复紧凑折叠：状态栏始终可见，处理进度和单阶段按钮默认收起，点击状态栏展开；完整工作区入口保持独立。

### 边界

- 本版本只修改界面布局、显示文案和正文状态区交互，不改变状态提取、总结、对象匹配、历史恢复或世界书逻辑。

## 1.3.0 — 2026-07-20

### 标准化

- 建立开箱即用的长期记忆标准，明确当前快照、内部事实、小总结、大总结、关系图谱和世界书的独立职责。
- 状态规则扩展为“允许建档 / 默认排除 / 证据与不确定性 / 更新与冲突”四段白盒配置。
- 默认规则允许不建档、不更新和保留不确定性，收紧背景板 NPC、模糊身份、推测事实与普通复述。
- 小总结和大总结核心判断更新为近期最小连续性与跨阶段长期资格，不再依赖字数区分。

### 表格与图谱

- 收紧默认表格用途，角色、物品、事件、区域、全局变化、基础设定和自定义对象分别具有明确准入边界。
- 对象表增加搜索、结果计数和固定关键列。
- 图谱优先使用明确关系、关联对象和关联事件；旧文本提及仅作为兼容回退并单独标记。
- 图谱增加搜索高亮、关系统计、节点连接度、图例和缩放，继续保持只读。

### UI

- 工作区、设置区和正文处理区移除折叠面板，关键规则与操作始终可见。
- 总览增加开箱检查、动态完成度和缺失项引导。
- 功能重新分配为总览、对象、规则、图谱、总结、审核、发布、设置和诊断。
- 普通页面减少固定协议、字段键、稳定 ID 和召回实现术语；完整 system prompt 与协议集中到只读诊断页。
- 增加实时状态反馈、搜索反馈和尊重 reduced-motion 的轻量动态效果。

### 兼容性

- 无破坏性数据结构变更。
- 旧设置自动补齐新增规则；稳定 ID、对象匹配、去重、历史恢复、聊天隔离和世界书事务边界保持不变。

## 1.2.1 — 2026-07-20

### 新增

- 表格管理页新增“对象准入提示词”，以“允许建档条件”和“默认排除条件”两块白盒文本控制新对象是否进入表格。
- 可查看当前完整状态提取提示词，并可一键恢复严格默认值。

### 调整

- 默认准入规则要求对象形成持续身份、独立因果作用、长期关系、状态、目标或未决事项后才建档。
- 路人、宾客、观众、侍卫、仆从、店员、士兵、村民和围观者等背景板 NPC 默认排除；姓名、服装、外貌、动作或一句台词不能单独构成建档依据。
- 后续正文真正赋予背景对象独立作用时，仍允许升级为正式条目。

### 边界

- 本次只开放模型的对象准入判断；固定文本协议、稳定 ID、匹配、合并、去重、历史保留和世界书写入继续由插件代码控制。
- 无数据结构破坏性变更；旧设置自动获得严格默认准入规则。

## 1.2.0 — 2026-07-20

### 新增

- 小总结和大总结提示词白盒化，分为核心判断、保留内容、排除内容、更新规则和表达规则。
- 总结页可查看完整规则提示词和固定输出协议，并可分别恢复默认值。
- GitHub 文档拆分为使用说明、插件逻辑、更新记录和正式版说明。

### 调整

- 小总结默认目标改为“近期必须承接的信息”。
- 大总结默认目标改为“跨场景、跨阶段仍不能遗忘的长期事实”，不再把小总结逐项缩写。
- 默认总结提示词移除身份型前缀，直接使用任务规格。
- 表头设置改为“表头名称｜记录要求”，不再要求玩家填写字段键和字段类型。
- 自定义表头创建、改名和排序时，插件自动维护内部稳定字段键；旧版高级格式继续兼容。
- README 不再堆叠全部候选版本报告。

### 修复与稳定性

- 阻止聊天 A→B→A 后旧活动任务恢复提交资格。
- 固定文本块缺少、嵌套或错配结束标签时明确失败，不提交半成品。
- 小总结空响应后释放队列键并允许真实重试。
- 历史重建中间失败后从失败消息续跑，不重复成功前缀。
- 世界书保存后使用单一可等待刷新链，避免界面显示旧条目。
- 同聊天、同表、同规范名称强制唯一对象身份和稳定 ID。
- 状态旧信息改为全局对象短目录与少量相关工作副本，降低大快照请求长度。

### 不兼容性

- 无数据结构破坏性变更。
- 旧设置会自动补充新的总结提示词默认配置。
- 旧表头高级格式仍可读取；新界面保存后使用简化显示格式。

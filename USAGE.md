# 镜渊使用说明

## 1. 第一次打开

安装并刷新 SillyTavern 后，打开镜渊工作区。先看“总览”中的开箱检查：它会显示插件启用、状态提取连接、总结连接、世界书发布、表格和默认规则是否已准备完成。

最少只需确认三件事：

1. 状态提取使用“当前连接”或一个可用的 Connection Profile；
2. 自动状态提取已开启；
3. 需要长期召回时，开启世界书发布。

审核、定向修正、小总结和大总结可以先沿用当前连接。只有在速度、费用或格式稳定性上有明确需求时，再分配不同模型。

## 2. 页面分工

- **总览**：查看开箱进度、当前聊天状态、最近任务和下一步操作。
- **对象**：浏览和编辑当前有效对象；支持搜索、按表查看和横向编辑。
- **规则**：编辑表格用途、表头要求、对象准入、证据边界以及两级总结规则。
- **图谱**：查看已经形成的角色关系、对象关联和事件关联；它是只读视图，不反向生成事实。
- **总结**：查看已生成的小总结、大总结及其事件槽位，不在此重复放置规则编辑器。
- **审核**：设置可验证的正文规则和修正方式。
- **发布**：把当前有效记忆发布到世界书，或按对象清理后重新发布。
- **设置**：配置自动化、显示、模型分工和维护参数。
- **诊断**：查看请求、错误、完整 system prompt 与固定协议；普通使用不需要打开。

所有主要页面始终可见，不依赖折叠区才能找到关键操作。

## 3. 正常处理流程

角色正文生成后，镜渊按顺序处理：

1. 审核正文；
2. 必要时定向修正；
3. 提取本轮事实和对象变化；
4. 更新当前聊天的完整状态快照；
5. 达到条件时生成小总结和大总结；
6. 将有效对象记忆发布到世界书。

正文下方处理区和“总览”都会显示当前阶段。派生阶段失败不会回滚已经成功保存的核心状态。

## 4. 标准对象准入规则

“规则”页把状态提取分成四块白盒规则：

- **允许建档条件**：什么对象值得进入长期记忆；
- **默认排除条件**：哪些背景板、临时对象和场景细节不应建档；
- **证据与不确定性**：如何处理传闻、冲突、同名对象和无法确认的信息；
- **更新与冲突**：什么算真实变化，如何替换旧状态、关闭未决事项和更新关系。

默认规则采用保守写入：姓名、服装、外貌、站位、一句台词或一次短动作都不能单独触发建档。路人、宾客、侍卫、店员、村民等背景角色默认忽略；后续真正形成持续关系、目标、责任、状态或因果作用时，仍可升级为正式对象。

遇到身份不清、证据不足或无法判断是否变化时，模型被要求省略，而不是猜测写入。

## 5. 小总结与大总结

小总结回答：

> 继续这条事件线的下一步，必须保留的最小事实集合是什么？

它维护近期事件连续性：必要起因、关键承诺、阶段结果、持续影响和下一步未决事项。

大总结回答：

> 即使近期过程全部遗忘，哪些已确认事实仍会跨场景、跨阶段改变未来？

它只固化长期结果、不可逆变化、长期关系/能力/归属变化、持续因果和跨阶段未决事项。

两者都不以固定字数区分。每套规则包含核心判断、保留内容、排除内容、更新规则和表达规则，并在“规则”页始终可见。

## 6. 表格与表头

默认表格已经带有严格用途，可直接使用。只有业务需要时再调整：

- 表格名称；
- 表格用途；
- 是否启用；
- 表格顺序；
- 基础表头；
- 附加表头。

表头格式为：

```text
表头名称｜记录要求
```

例如：

```text
战斗风格｜只记录角色长期稳定并会影响后续战斗的方式、武器偏好和限制
```

内部字段键、字段类型、稳定身份和历史映射由插件维护。自定义表不是杂项垃圾桶，也应写明对象资格和字段边界。

## 7. 记忆图谱

图谱优先使用表格中的明确关联：

- 关系状态；
- 关联对象；
- 关联事件。

旧记录没有这些字段时，才使用文本提及作为兼容连线，并标记为“旧记录关联”。普通同场、一次对话或背景描写不会自动建立新关系。

图谱提供搜索、高亮、关系类型图例、节点连接度和缩放。图谱只用于浏览和排错，不是新的事实来源。

## 8. 焦点对象

每个聊天最多设置一个角色焦点。焦点只让对应世界书条目成为常驻条目，不改变对象排序、容量裁剪或其他召回逻辑。

## 9. 世界书发布

“发布”页提供两个操作：

- **立即发布**：按当前真实快照更新托管条目；
- **按对象整理并重新发布**：清理当前聊天可安全识别的旧镜渊条目，再重新发布。

人工条目、其他聊天条目和无法确认归属的共享书条目不会被普通维护删除。

界面使用“始终带入 / 关键词触发 / 语义召回”等用户语言解释发布方式，底层 constant、trigger、vector、事实 ID 和事件 ID 不在普通页面暴露。

## 10. 历史编辑与恢复

编辑、删除或 Swipe 历史 AI 正文后，镜渊会标记最早失效位置，并暂停世界书发布。

累计重建按消息顺序执行：

- 失效点之前的成功快照继续使用；
- 每成功一条立即保存检查点；
- 中间失败后，从失败消息继续；
- 核心状态已完成时，不会因为总结或世界书失败而重新提取全部状态。

## 11. 常见问题

### 表格出现背景板 NPC

先在“规则”页恢复标准规则，再检查角色表用途是否被改得过宽。已有错误条目不会自动删除；修正规则后手动删除或从受影响历史位置重建。

### 请求很慢

打开“诊断”查看耗时。TaskQueue 只有几毫秒而上游响应很长时，主要瓶颈通常在模型服务，不是表格或图谱。

### 返回 HTML、空响应或缺少结束标签

镜渊会明确标记失败并给出阶段信息，不会提交半成品。可在诊断页复制该任务的完整 system prompt 对照模型响应。

### 小总结和大总结信息量接近

检查两者的核心判断，而不是单纯限制字数。小总结保留下一步连续性；大总结只保留跨阶段仍然改变未来的已确认事实。

### 手机表格很宽

关键列固定在左侧，其他列在表格区域内横向滑动。页面本身不会横向溢出。

# Mirror Abyss rc.43 — 已覆盖小总结重复显示修复

## 实机现象

rc.42 的“分层总结”页面同时显示：

- 小总结：`沉秋在陈家面馆的对峙`
- 大总结：`沉秋在陈家面馆的对峙`

大总结已经成功固化该事件线，小总结也已写入 `solidifiedByLargeSummaryId`，但 UI 仍直接渲染 `chatState.smallSummaries` 的全部历史记录，因此被覆盖退出的小总结继续作为活动条目显示。

## 根因

总结生成与消费逻辑正确：

- 大总结生成后会标记被消费的小总结；
- 被标记的小总结不会再次参与大总结生成；
- 世界书语义模式不会把小总结和大总结分别发布为独立世界书条目。

错误仅位于 `src/ui/workspace.ts`：`summariesHtml()` 没有使用已有的 `pendingSmallSummaries()` 活动版本筛选器。

## 修复

“分层总结”页面现在仅显示：

- 未被大总结固化的小总结；
- 未被新版小总结替代的小总结；
- 未被旧格式大总结消费的小总结。

历史小总结仍保留在内部数据中，用于追溯、迁移和关系链校验，不做删除或压缩。

## 未修改

- 大总结触发规则；
- `largeSummaryCount`；
- 小总结/大总结数据结构；
- 世界书发布结构；
- 状态提取、审核、队列与历史恢复；
- 已有总结内容。

## 回归

新增 `tests/rc43-summary-display.ts`，覆盖：

1. 已固化的小总结不再作为活动条目显示；
2. 被新版替代的小总结不再显示；
3. 被旧格式大总结消费的小总结不再显示；
4. 未消费的小总结正常保留；
5. UI 必须接入活动小总结筛选器。

完整 `npm run verify`、TypeScript、浏览器构建、语法检查及 20 步玩家流程均通过。

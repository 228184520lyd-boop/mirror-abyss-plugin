# Mirror Abyss 1.2.0 正式版

`1.2.0` 是镜渊第一版稳定发布。它冻结了固定文本通信、对象身份、状态快照、分层总结、历史恢复和世界书发布的主要业务边界。

## 本次正式版变化

### 分层总结提示词公开调整

小总结和大总结不再依靠隐藏在源码中的完整长提示词。两套规则均可在总结页直接查看和编辑：

- 核心判断；
- 保留内容；
- 排除内容；
- 更新规则；
- 表达规则。

固定输出协议保持只读可见。插件不会额外创建测试任务或诊断存储；下一次真实总结就是新提示词的实际验证。

### 小总结和大总结职责分离

小总结维护近期事件连续性，大总结执行长期资格审查。系统不以硬字数作为主要规则，也不要求大总结机械压缩为固定比例。

### 表头设置简化

表头编辑只填写：

```text
表头名称｜记录要求
```

内部 key、类型和旧数据兼容由插件管理。旧高级格式仍然可读，避免已有设置失效。

## 升级方式

可以直接使用 `Mirror-Abyss-1.2.0.zip` 覆盖旧扩展。刷新 SillyTavern 后，现有聊天快照、内部事实、总结版本和世界书映射继续使用。

升级后建议检查：

1. 总结页的小总结和大总结默认规则；
2. 表格页的自定义表头显示；
3. 当前聊天世界书同步状态；
4. 正在进行的历史重建是否按原检查点继续。

## 已知边界

- 未声称所有厂商定制 Android 浏览器均已实体设备验证。
- HTML 网关错误不会污染状态，但传输 trace 可能先记录已收到文本，再在固定文本解析阶段失败。
- 总结质量仍受所选模型的指令遵循与语言理解能力影响；白盒提示词用于直接调整其信息准入标准。
